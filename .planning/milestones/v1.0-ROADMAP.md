## M5 详细定义

### 目标
在运维操作场景中嵌入 AI 分析能力，由 agent 主动采集信息并给出分析结论。触发方式：事件触发 + 手动触发。

### 全局约束
| 约束 | 说明 |
|------|------|
| 只读账号 | 系统纳管数据库仅使用只读账号，不支持 DDL 执行 |
| 无 OS 访问 | 没有操作系统级账号，无法采集 OS 日志 |
| LLM 可用 | SQL 审核和 AI 分析可使用 LLM 能力（Anthropic/OpenAI/Ollama） |
| M4 前提 | TopSQL AI 分析依赖 M4.4 索引采集数据 |
| AI 触发方式 | 事件触发 + 手动触发，异步执行，不阻塞主流程 |

### 阶段划分

| 阶段 | 名称 | 预计工作量 | 依赖 |
|------|------|-----------|------|
| 05 | AI 增强（5 合一）| 15-20 小时 | 依赖 M4 全部 |

### Phase 05 详细定义：AI 增强

合并 5 个子场景为单一 Phase 05 统一执行：

1. **TopSQL AI 分析** — 慢查询自动分析，给出优化建议
2. **告警根因分析** — 告警触发后 AI agent 自动采集数据并分析根因
3. **故障自动诊断** — 健康评分下降或手动触发时，完整诊断流程
4. **前端 AI 分析入口** — 手动触发按钮 + 轮询结果展示
5. **容量预测** — 基于 metrics_history 的多指标线性回归预测

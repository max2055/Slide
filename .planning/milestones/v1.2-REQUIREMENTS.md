# Requirements — v1.2

## UI-01: UI风格一致化

统一所有页面的字体、图标尺寸、卡片间距、色彩变量使用。制定 UI 规范并应用到全局。

**Acceptance:**
- 所有页面使用统一的 CSS 变量（font-size, icon-size, spacing, radius）
- 表格、卡片、按钮、badge 样式一致
- 暗色/亮色主题切换流畅
- 响应式布局在所有页面生效

## UI-02: 菜单精简

清理 Slide 项目不需要的 OpenClaw 平台管理菜单和设置项。

**Acceptance:**
- 移除 Slide 不需要的 tab：sessions, usage, skills, config, appearance, system
- 保留 Slide 核心功能：overview, cron, agents, users, llm-config, rbac
- 移除后无死代码残留（type、路径、事件监听）

## AI-01: AI 分析结果可见

自动 AI 分析（RCA、TopSQL、故障诊断）的结果在告警列表和实例详情中可见。

**Acceptance:**
- 告警列表显示"已分析"标记 + 可点击查看分析结果
- 实例详情页显示最近 AI 诊断结果摘要
- 分析结果存储与展示格式统一

## AI-02: AI Agent 运维助手

在 Chat 中增强数据库运维上下文，让 AI 能读取实例状态、告警、慢查询并给出建议。

**Acceptance:**
- Chat 对话中 AI 可回答"哪些实例有问题""最近的慢查询是什么"等问题
- 运维事件通知到聊天频道

## DOC-01: 项目文档

编写完整的项目文档。

**Acceptance:**
- ARCHITECTURE.md — 系统架构、模块关系、数据流
- OPERATIONS.md — 运行机制、配置项、启停流程
- USER-GUIDE.md — 用户手册（各功能模块使用说明）
- docs/ 目录结构清晰

## DB-01: 达梦数据库纳管支持

将达梦（Dameng）数据库纳入管理平台，支持实例连接、SQL 控制台、指标采集、慢查询分析。

**Acceptance:**
- 实例管理中可添加达梦数据库实例
- SQL 控制台支持达梦 SQL 方言（语法高亮、自动补全、EXPLAIN）
- 指标采集支持达梦（连接数、QPS、容量等）
- 实例详情页显示达梦特有信息（表空间、模式）

## DB-02: Oracle 数据库纳管支持

将 Oracle 数据库纳入管理平台，支持实例连接、SQL 控制台、指标采集。

**Acceptance:**
- 实例管理中可添加 Oracle 数据库实例
- SQL 控制台支持 Oracle SQL 方言（PL/SQL 语法高亮、自动补全）
- 指标采集支持 Oracle（SGA/PGA、表空间、会话等）
- 实例详情页显示 Oracle 特有信息

## SQLC-01: SQL 执行历史持久化

将 SQL 控制台的执行历史从内存存储改为数据库持久化，确保服务器重启后历史不丢失。

**Acceptance:**
- 新建 `sql_execution_history` 表存储执行记录（SQL文本、实例ID、数据库名、耗时、行数、状态、时间戳）
- SQL 执行时自动写入历史表
- 查询历史 API 从数据库表读取（支持分页、搜索、筛选）
- 内存缓存层可保留作为热数据加速

## MISC-01: OpenClaw upstream 合并


将上游 OpenClaw 的安全修复和 gateway 稳定性更新合并到 Slide。

**Acceptance:**
- upstream/main 合并到 slide-custom，冲突解决
- 核心功能（gateway, chat, tools）正常
- 合并后无回归

## BUG-01: 数据库连接自动恢复

纳管数据库断开重连后，Slide 自动检测连接状态并重建连接，实例健康状态随数据库状态实时同步。

**Acceptance:**
- 数据库断开后下次查询时自动检测到连接失效并触发重连
- 重连成功后实例 health_status 恢复为 healthy
- 数据库持续不可达时实例保持在 critical 状态
- MySQL pool 配置 enableKeepAlive 减少意外断开
- 执行 SQL 时若连接断开自动重连后重试（最多一次）

# Requirements: Slide — AI 驱动的数据库运维平台

**Defined:** 2026-05-09
**Core Value:** AI 原生的数据库运维 — Agent 自动采集数据、分析问题、给出建议

## v1.1 Requirements

Requirements for v1.1 功能完善 milestone. Each maps to roadmap phases.

### RBAC

- [x] **RBAC-01**: Admin can create custom roles with name and description
- [x] **RBAC-02**: Admin can define permission codes following `resource:action` namespace convention
- [x] **RBAC-03**: Admin can assign multiple permissions to a role (many-to-many)
- [x] **RBAC-04**: Admin can assign multiple roles to a user (many-to-many), replacing single ENUM role
- [x] **RBAC-05**: Admin can grant user access to specific instances (instance-level resource control)
- [x] **RBAC-06**: `requirePermission` middleware checks role-based permissions before route handlers execute
- [x] **RBAC-07**: `requireInstanceAccess` middleware checks user-instance binding on all instance-scoped routes
- [x] **RBAC-08**: Existing users are migrated from `users.role` ENUM to `user_roles` table without permission loss
- [x] **RBAC-09**: Frontend provides role/user/permission/instance-access CRUD management page

### Approval

- [x] **APPR-01**: Approval detail page shows SQL with syntax highlighting in read-only CodeMirror editor
- [x] **APPR-02**: Detail page shows full approval history timeline (submit/review/execute/notify events)
- [x] **APPR-03**: Approver can batch select multiple pending requests and approve/reject in one action
- [x] **APPR-04**: Approve action optionally auto-executes SQL, recording execution result on the approval record
- [x] **APPR-05**: Reject action writes reviewer comment back to approval request
- [x] **APPR-06**: Approval outcome notifies the requester via existing notification channels (DingTalk/WeCom/Feishu/Webhook)

### SQL Console

- [x] **SQLC-01**: SQL editor provides schema-driven autocomplete (tables, columns) via @codemirror/autocomplete
- [x] **SQLC-02**: Result table supports column header click to sort (ascending/descending/none)
- [x] **SQLC-03**: Result table supports client-side pagination with configurable page size
- [x] **SQLC-04**: User can export query results to CSV via download button
- [x] **SQLC-05**: User can open multiple query tabs, each with independent editor state and result set
- [x] **SQLC-06**: Query history is automatically saved (SQL text, instance, duration, row count) and searchable
- [x] **SQLC-07**: EXPLAIN JSON output is rendered as a visual tree/table component in the console

### Dashboard

- [x] **DASH-01**: Dashboard shows DB type distribution as ECharts pie/bar chart (MySQL/PostgreSQL counts and ratios)
- [x] **DASH-02**: Dashboard shows total data volume trend as ECharts line chart (aggregated from capacity_history)
- [x] **DASH-03**: Dashboard shows instance health status summary cards (healthy/warning/critical/offline counts)
- [x] **DASH-04**: Dashboard card layout is reorganized with CSS Grid (stat cards row → charts row → instance list)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Monaco Editor | CodeMirror 6 already integrated, Monaco adds 2-3MB bundle without additional benefit |
| Casbin policy engine | Overkill for <100 roles, custom RBAC on existing pattern is simpler |
| XLSX export | CSV opens fine in Excel, add later if users demand |
| Drag-and-drop dashboard layout | CSS Grid reordering sufficient for v1.1, runtime drag adds disproportionate complexity |
| OAuth/SSO login | Email/password + JWT sufficient for internal tool |

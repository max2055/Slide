## Integration Findings

### Critical Blockers

**B1: Agent tools not registered at runtime**

All 11 agent tool files call `toolCatalog.register()` at module level, but none are imported by any production entry point (`server.ts`, `agent-service.ts`, etc.). The tools are never registered, so `toolCatalog.getAll().filter(t => t.group === 'db_ops')` returns an empty array.

This breaks:
- Chat AI cannot list alerts or get instance summaries (AI-02)
- Agent cannot save AI analysis results (AI-01)
- Agent cannot add or test Dameng/Oracle connections (DB-01, DB-02)
- Agent cannot generate ASH/AWR/tablespace reports (DB-02)

**B2: Documentation doesn't cover Dameng or Oracle**

Phase 94 docs (ARCHITECTURE.md, USER-GUIDE.md, OPERATIONS.md) were written 2026-05-17, before Dameng (Phase 95, 2026-05-18) and Oracle (Phase 96, 2026-05-19) completed. Zero mentions of Dameng in any doc; Oracle mentioned only in generic architecture context.

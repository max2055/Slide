## Copywriting Contract

### Add-Instance Form: Oracle Database Identifier Field

| Element | Copy |
|---------|------|
| Label | Oracle 数据库标识 |
| Placeholder | 输入 SID 或 Service Name |
| Hint | 支持 SID 或 Service Name，格式：host:port/标识。默认 SID: ORCL |
| Visibility | Show ONLY when db_type === 'oracle'. Hidden for all other DB types. |
| Position | After the "端口" row, before "主机地址". Same form-group pattern as existing fields. |

### Instance Detail Overview Tab — Oracle-Specific Section

New conditional section inside `_renderOverview()`, shown only when `db_type === 'oracle'`. Uses the same `overview-item`/`overview-grid` pattern as existing DB-specific metrics (see L1871-1904 for the MySQL InnoDB pattern reference).

| Stat Item | Label | Data Source | Unit / Format |
|-----------|-------|-------------|---------------|
| Oracle Version | Oracle 版本 | `this.metrics.version` (string from V$VERSION) | Free text, e.g. "Oracle Database 19c EE" |
| SGA Size | SGA 大小 | `this.metrics.sga_size_gb` | `X.XX GB` |
| PGA Size | PGA 大小 | `this.metrics.pga_size_gb` | `X.XX GB` |
| Tablespace Count | 表空间数 | `this.capacity.tablespaces?.length` | `N 个` |
| Tablespace Usage | 表空间使用率 | Aggregate from `this.capacity.tablespaces[]` | `XX.X%` (avg usage with color coding) |

### Empty States (Oracle-specific)

| Condition | Message |
|-----------|---------|
| Metrics not yet collected (SGA/PGA/version = undefined) | Display `—` as overview value. No separate empty state — the overview item renders with placeholder dash. |
| Capacity/tablespace data not yet collected (tablespaces = null/undefined) | Display `—` for tablespace count and usage. |
| SHOW ONLY when metrics block exists (`this.metrics` is truthy) | Gate the entire Oracle-specific card behind `db_type === 'oracle' && this.metrics` |

### Connection Test Failure (Oracle-specific)

Uses the existing test connection pattern at L1378-1391 of `instances-db.ts`. The `_handleTestConnection` already handles all db_types. Oracle connection errors from the backend are displayed inline via the existing `testMessage` state.

| Status | Pattern |
|--------|---------|
| Testing | `${icons.loader} 测试中...` |
| Success | `${icons.checkCircle} 连接成功` (green) |
| Error | `${icons.xCircle}` + error message from server (red) |

### Error State (SQL Console)

Uses the existing `.error-box` pattern at L213 of `sql-console.ts`. Oracle-specific error messages (e.g. "PLAN_TABLE does not exist", "ORA-xxxxx") display inline in the error box below the editor. No custom Oracle error formatting needed — the existing error display handles all db_types uniformly.

### Destructive Actions

None. Phase 96 has no destructive actions — all changes are additive (new form field, new SQL dialect, new overview cards). No delete, remove, or destructive operations.

## Interaction Contracts

### 1. SQL Console: OracleDialect Selection

**Trigger:** User selects an Oracle instance from the instance dropdown.

**Interaction:**
1. Instance dropdown change fires event handler
2. `selectedDialect` resolves: `instance?.db_type === 'oracle' ? OracleDialect : instance?.db_type === 'dameng' ? DamengDialect : MySQL`
3. `_switchTab()` creates a new EditorView with OracleDialect passed to `sql({ dialect: selectedDialect })`
4. Autocomplete source (`_sqlCompletions`) provides Oracle-specific completions (system views, functions, PL/SQL keywords)

**Behavior on existing tabs:** Tab editor views are created once. If user switches instance while tabs are open, existing tabs keep their original dialect. New tabs use the current selected instance's dialect.

**No special keyboard shortcuts** — same as existing SQL Console (Ctrl+Enter to execute).

### 2. Add-Instance Form: Oracle Database Identifier Field

**Trigger:** User selects "Oracle" from the db_type dropdown.

**Interaction:**
1. On db_type change, if `db_type === 'oracle'`, show the database identifier field
2. If db_type changes away from 'oracle', hide the field
3. The field value is stored in the form's connection config JSON on the backend
4. Default port auto-changes to 1521 (already implemented at L1299-1302)

**Field behavior:**
- Plain `<input>` with `class="form-input"` (same as existing fields)
- No validation on input — the identifier is included in the connect string as-is
- No select/dropdown — single text input accepts SID or Service Name
- The existing "数据库名" (database_name) field still shows for Oracle — serves as default schema/database context

### 3. Instance Detail: Oracle-Specific Overview Cards

**Trigger:** User navigates to instance detail page for an Oracle instance (`this.instance.db_type === 'oracle'`).

**Interaction:**
1. Overview tab renders standard instance info card + metrics card
2. If `this.metrics` is available AND `db_type === 'oracle'`, render additional Oracle-specific card below metrics
3. Use existing `overview-grid` (4-column) layout pattern
4. Items from D-08: version, SGA size, PGA size, tablespace count + usage
5. Tablespace usage percentage uses conditional color: `usage_percent > 90 ? 'danger' : usage_percent > 70 ? 'warn' : 'ok'`
6. If `this.capacity` is not yet loaded, tablespace items display `—`

**Layout:** Exactly matches the MySQL InnoDB-specific card pattern (L1871-1904). Same `.card` > `.card-header` + `.card-title` > `.card-body` > `.overview-grid` > `.overview-item` structure.

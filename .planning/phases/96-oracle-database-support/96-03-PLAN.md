### Fix B: WR-02 — DBA tablespace query try/catch in checkOracleHealth() (lines 1691-1698)

The tablespace usage query at lines 1691-1698 is not wrapped in an inner try/catch, unlike getOracleMetrics() (lines 891-905) and getOracleCapacity() which both have proper fallback.

**Replace lines 1691-1699:**

Before:
```
      const tablespaceResult = await conn.oracleConnection.execute(`
        SELECT ROUND(SUM(bytes_used) / SUM(bytes_total) * 100, 2) as usage_percent
        FROM (
          SELECT SUM(bytes) as bytes_used, 0 as bytes_total FROM DBA_SEGMENTS
          UNION ALL
          SELECT 0, SUM(bytes) FROM DBA_DATA_FILES
        )
      `);
      const tablespaceUsage = tablespaceResult.rows[0]?.[0] as number || 50;
```

After:
```
      let tablespaceUsage = null;
      try {
        const tablespaceResult = await conn.oracleConnection.execute(`
          SELECT ROUND(SUM(bytes_used) / SUM(bytes_total) * 100, 2) as usage_percent
          FROM (
            SELECT SUM(bytes) as bytes_used, 0 as bytes_total FROM DBA_SEGMENTS
            UNION ALL
            SELECT 0, SUM(bytes) FROM DBA_DATA_FILES
          )
        `);
        tablespaceUsage = tablespaceResult.rows[0]?.[0] as number || null;
      } catch {
        console.warn(`[OracleHealth] DBA 视图权限不足，表空间使用率不可用`);
        tablespaceUsage = null;

```

Then update the health check logic after line 1699 (now after the try/catch block). The existing code at lines 1701-1703 does:

```
let tsStatus = 'ok';

let tsMessage = `表空间使用率：${tablespaceUsage.toFixed(1)}%`;
```

Since tablespaceUsage may now be null, change this to:
```
let tsStatus = 'ok';
let tsScore = 100;
let tsMessage: string;
if (tablespaceUsage === null) {
  tsMessage = '表空间使用率：不可用（DBA 权限不足）';
} else {
  tsMessage = `表空间使用率：${tablespaceUsage.toFixed(1)}%`;
  if (tablespaceUsage > 90) {
    tsStatus = 'critical';
    tsScore = 40;
    tsMessage = `表空间使用率过高：${tablespaceUsage.toFixed(1)}%`;
    totalScore -= 30;
  } else if (tablespaceUsage > 80) {
    tsStatus = 'warning';
    tsScore = 70;
    tsMessage = `表空间使用率较高：${tablespaceUsage.toFixed(1)}%`;
    totalScore -= 15;
  }
}
```

This pattern ensures that when DBA privileges are insufficient, the health check gracefully reports "unavailable" instead of returning health_score=0 / status=critical.

Fix null-unsafe `tablespace_usage_percent` check on lines 1767 and 1771 of `instance-detail.ts`.

The current check `!== undefined` does not catch `null`. When the backend returns `null` (e.g., DBA privileges insufficient), `null !== undefined` evaluates to `true`, passing the guard and then immediately crashing on `.toFixed(1)`.

**Line 1767 change — guard condition:**
Before: `${this.metrics.tablespace_usage_percent !== undefined ? html\``
After:  `${this.metrics.tablespace_usage_percent != null ? html\``

`!= null` in JavaScript/TypeScript checks for both `null` and `undefined` (it is the loose equality check that treats null and undefined as equal).

**Line 1771 change — null-safe toFixed:**
Before: `${this.metrics.tablespace_usage_percent.toFixed(1)}%`
After:  `${(this.metrics.tablespace_usage_percent ?? 0).toFixed(1)}%`

The `?? 0` provides a fallback value of 0 in case the value is somehow null/undefined despite the guard (defensive coding).

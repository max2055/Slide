## Interactions

### RPT-02: Scheduled Configs Card

**Layout position:** Between the stat-card row (lines 426-431) and the "Generate Report" card (line 434).

**Card structure:**
```
+--------------------------------------------------+
| [calendar icon] 定时报表配置    [刷新] [新增配置] |
| 设置定时报表自动生成任务                           |
+--------------------------------------------------+
| [empty state or table]                            |
|                                                   |
| Table columns:                                    |
| 配置名称 | 目标实例 | 报表类型 | Cron 表达式 |    |
| 下次执行 | 状态[启用/停用] | 操作[编辑/删除]     |
|                                                   |
+--------------------------------------------------+
```

**Interactions:**

1. **Load:** On `firstUpdated()`, fetch `GET /api/reports/configs` and render config list or empty state.
2. **Refresh:** Click refresh icon button in card header to reload config list. Show loading indicator during fetch.
3. **Add config:** Click "新增配置" button opens create modal/dialog.
4. **Edit config:** Click "编辑" action button opens edit modal/dialog pre-filled with config data.
5. **Delete config:** Click "删除" action button shows a confirmation dialog. On confirm, `DELETE /api/reports/configs/:id`. Show success feedback via the alert() pattern (consistent with existing code) or inline toast.
6. **Toggle enable/disable:** Click toggle switch sends `PUT /api/reports/configs/:id` with `{ enabled: !currentValue }`. The switch must optimistically toggle in the UI before the server responds (with rollback on error). Use the existing `.cfg-toggle` CSS pattern from alerts.ts lines 654-697.
7. **View next run time:** Computed on the frontend using `new CronJob(config.cron).nextDates(1)` after fetching the config. Display as relative time (e.g. "5 小时后") or absolute ("2026-05-22 03:00").
8. **Empty state:** Display centered empty state with `file-text` icon when no configs exist.

**Create/Edit Dialog:**

```
+-----------------------------------+
| 新增定时报表配置             [x]  |
+-----------------------------------+
| 配置名称: [____________________]  |
| 报表类型: [dropdown: health/perf..]|
| 目标实例: [dropdown: instance list]|
| Cron 表达式: [________________]   |
| 输出格式: [dropdown: HTML/PDF/..] |
|                                    |
| [启用] toggle switch               |
|                                    |
|        [取消] [创建]              |
+-----------------------------------+
```

- The dialog should use the same modal pattern as the existing config panel in alerts.ts (`.cfg-toggle-row`, `.modal--config` classes).
- Width: `max-width: 600px` for the dialog.
- Input styling matches existing `<select>` at line 442-450.
- Submit button uses `.action-btn.primary` style.
- Cancel button uses `.action-btn` style.
- Close on escape key and click-outside (consistent with existing modal pattern).

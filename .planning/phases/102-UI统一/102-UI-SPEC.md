## Spacing Scale

The project already defines design tokens in `base.css`. These are listed for reference — this phase does not introduce new spacing tokens.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, inline padding within stat-card |
| `--space-sm` | 8px | Compact element spacing |
| `--space-md` | 12px | Default element spacing within cards |
| `--space-lg` | 16px | Card padding (stat-card uses `padding: 16px`) |
| `--space-xl` | 24px | Section padding, card gap in grid layouts |

**Note:** `--space-md` is 12px (not a multiple of 4). This is an existing project convention — do not change existing tokens.

**stat-card internal layout (extracted from existing ov-card CSS):**
- Card padding: 16px (`--space-lg`)
- Internal grid gap: 6px (between label/value/hint rows)
- Default card gap in grid: 24px (`--space-xl`)

**Responsive:** At narrow widths (`--mobile` breakpoint), card padding shrinks to 12px.

**Exceptions for this phase:**
- **None.** The stat-card component inherits all spacing from existing ov-card CSS tokens. No custom spacing values needed.

## Typography

The project defines 7 font-size tokens. This phase uses 3 for the stat-card component (migrated from existing ov-card CSS) and existing body defaults for surrounding text.

### Existing Token Set (for reference)

| Token | Size | Usage |
|-------|------|-------|
| `--text-xs` | 11px | Small labels, secondary metadata |
| `--text-sm` | 12px | Hint text, compact UI |
| `--text-base` | 13px | (reserved, not actively used) |
| `--text-md` | 14px | **Body text (default)** |
| `--text-lg` | 16px | Emphasized body, navigation items |
| `--text-xl` | 18px | Card value (responsive) |
| `--text-2xl` | 22px | Card value (desktop) |

### Contract for Phase 102

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 14px (`--text-md`) | 400 (Regular) | 1.55 | Default body — unchanged by this phase |
| stat-card label | 11px (`--text-xs`) | 600 (Semibold) | inherit | From existing `.ov-card__label` pattern |
| stat-card value | 22px (`--text-2xl`) | 700 (Bold) | inherit | Responsive: shrinks to 18px on narrow widths |
| stat-card hint | 12px (`--text-sm`) | 400 (Regular) | 1.35 | From existing `.ov-card__hint` pattern |
| icon size | 16px (`--text-lg`) | — | — | Default icon container size, set via CSS `.icon` class |

**Font weights in use:** 400 (Regular), 600 (Semibold), 700 (Bold). The stat-card value uses weight 700 — this is an existing pattern from ov-card CSS and is preserved in the new component.

**Font family:** `var(--font-body) = "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` for all UI text. `var(--mono)` for code/monospace contexts.

## Color

The project has a comprehensive theme system with light and dark modes. All color values are defined as CSS variables in `global.css` (Slide theme) and `base.css` (OpenClaw theme, syncs at load).

This phase does not touch theme colors. The stat-card component references existing CSS variables.

### Contract

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant surface (60%) | `--bg` (#f8f9fa light, #111113 dark) | Page background, surrounding layout |
| Secondary surface (30%) | `--card` (#ffffff light, #1c1c1f dark) | stat-card background, popovers, panels |
| Accent (10%) | `--accent` (#409eff) | Interactive elements: links, focus rings, primary buttons, active states, selection |
| Destructive | `--destructive` (#dc2626) | Destructive actions only: delete buttons, error indicators, critical variant |
| Semantic OK | `--ok` (#15803d) | stat-card `variant="ok"` indicator dot, success status |
| Semantic Warning | `--warn` (#b45309) | stat-card `variant="warn"` indicator dot, warning status |
| Semantic Info | `--info` (#2563eb) | stat-card `variant="info"` indicator dot, informational status |
| Text | `--text` / `--text-strong` | Body text and headings — stat-card label/value/hint use these |
| Muted | `--muted` | stat-card label and hint text color |
| Border | `--border` (#e5e5ea light) | stat-card border (1px solid `var(--border)`) |
| Card radius | `--radius-lg` (14px) | stat-card border-radius (existing ov-card value) |

### Accent reserved for
Interactive controls: links, buttons, focus rings, active tab indicators, primary CTAs, selected states, chart highlights. **Not used for** decorative backgrounds (use `--accent-subtle` instead) or general text.

### Variant colors (stat-card specific)
stat-card variant indicator dots use the four semantic colors defined above. These are the ONLY new visual elements affected by this phase — the indicator dot is a small circle rendered via CSS `::before` or similar, colored by the variant CSS variable.

## Phase-Specific Design Contract

### Icons (`frontend/src/icons.ts`)

| Property | Specification |
|----------|---------------|
| File location | `frontend/src/icons.ts` (new file) |
| Source files | Merge `openclaw/ui/icons.ts` + `styles/icons.ts`, then delete both |
| Naming convention | kebab-case (Lucide style), e.g. `eye-off`, `bar-chart`, `check-circle` |
| SVG attributes required | `fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"` |
| API exports | `icons`, `icon(name)`, `renderIcon(name, className?)`, `IconName` type |
| `renderIcon` default className | `'icon'` (matches `styles/icons.ts` convention) |
| Dedup priority | `styles/icons.ts` name wins for same-graphic conflicts |
| Both-graphically-different | Keep both (e.g. `check` and `circle-check` coexist) |
| Camouflage handling | Convert camelCase to kebab-case during merge — no alias transition layer |
| New icons needed | `bell-off`, `party-popper`, `calendar`, `pause`, `alert-circle`, `x-circle`, `message-square`, `layout-dashboard` (add from Lucide SVG source or create minimal equivalents) |
| Total icon count | ~70+ after merge |

### `<stat-card>` Component

| Property | Specification |
|----------|---------------|
| File location | `frontend/src/components/stat-card.ts` |
| Base class | `StatCard extends LitElement` |
| Custom element tag | `<stat-card>` |
| Registration | `customElements.define('stat-card', StatCard)` at bottom of file |
| Pattern reference | `AppHeader.ts`, `AppSidebar.ts` (existing LitElement pattern in this project) |

**Properties:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | `''` | Primary label displayed at top of card |
| `value` | `string` | `''` | Large numeric/statistical value |
| `hint` | `string` | `''` | Secondary hint text below value |
| `variant` | `'ok' \| 'warn' \| 'danger' \| 'info' \| undefined` | `undefined` | Renders colored indicator dot; no variant = no dot |

**CSS variable mapping:**

| Card part | CSS var | Fallback |
|-----------|---------|----------|
| Background | `var(--card)` | #ffffff |
| Border | `var(--border)` | #e5e5ea |
| Radius | `var(--radius-lg)` | 14px |
| Label color | `var(--muted)` | #6e6e73 |
| Value color | `var(--text-strong)` | #1a1a1e |
| Hint color | `var(--muted)` | #6e6e73 |
| OK dot | `var(--ok)` | #15803d |
| Warn dot | `var(--warn)` | #b45309 |
| Danger dot | `var(--destructive)` | #dc2626 |
| Info dot | `var(--info)` | #2563eb |

**Interaction contract:**
- Pure display — no built-in `@click` handler
- No icon slot or icon support
- Parent wraps in `<button>` for clickability (existing pattern in `overview-cards.ts`)
- No focus trap, no keyboard interaction — not interactive
- `part="root"` attribute on host element for external CSS customization

**Light DOM vs Shadow DOM:**
- Styled via Shadow DOM (standard LitElement)
- CSS variables penetrate Shadow DOM automatically — no `:host-context` or `@apply` needed
- Exposes `part="root"` for external styling overrides

**Responsive behavior:**
- At narrow widths (`@media (max-width: 640px)` or similar), value font shrinks from 22px to 18px (existing ov-card pattern)
- Card padding shrinks from 16px to 12px

4. Delete `openclaw/ui/icons.ts` **AFTER** all imports point to `src/icons.js`
5. Delete `styles/icons.ts` **AFTER** all imports point to `src/icons.js`

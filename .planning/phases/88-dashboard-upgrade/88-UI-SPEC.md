## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| 3xs | 4px | Icon-to-text gaps, badge inner padding |
| 2xs | 8px | Stat card internal label-to-value gap, health card icon-to-text gap |
| sm | 12px | Card grid column gap, quick action gap, bottom panel gap |
| md | 16px | Card padding (inner), chart card padding, section top/bottom margin |
| lg | 20px | Dashboard grid section gap (between rows) |
| xl | 24px | Page-level padding bottom |
| 2xl | 32px | Large component internal padding (empty states) |

Exceptions: none

**Default body line-height** is 1.55 (inherited from base.css `body`). All spacing on 4px baseline.

**Source:** Existing `dashboard.ts` patterns (`.ov-cards { gap: 12px }`, `.ov-card { padding: 16px }`, `.dashboard { padding: 0 0 24px 0 }`), CONTEXT.md D-22 layout.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 700 | 1.2 | Health status card counts (healthy/warning/critical/offline numbers) |
| Heading | 22px | 700 | 1.2 | Stat card values (instance count, data volume, alert count, AI analysis count) |
| Body | 14px | 400 | 1.55 | Default text, chart titles, health status labels, action card titles |
| Label | 12px | 600 | 1.4 | Stat card label text, subtitles, card hints, badge text — `text-transform: uppercase`, `letter-spacing: 0.04em`, color `var(--muted)` |

Additional implicit size:
- 13px at weight 500 for instance names in health list items (existing, kept as-is)
- 11px at weight 600 for existing badge text (existing, kept as-is)

**Source:** `frontend/src/openclaw/styles/base.css` (body: `font: 400 14px/1.55 var(--font-body)`), existing `dashboard.ts` (`.ov-card__value { font-size: 22px; font-weight: 700 }`, `.ov-card__label { font-size: 11px; font-weight: 600 }`), existing UI-SPEC typography.

## Color

| Role | Value (Light) | Value (Dark) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `var(--card)` (#ffffff), `var(--bg)` (#f8f9fa), `var(--border)` (#e5e5ea) | `var(--card)` (#161920), `var(--bg)` (#0e1015), `var(--border)` (#1e2028) | Page surface, card backgrounds, structural dividers |
| Secondary (30%) | `var(--text)` (#3c3c43), `var(--muted)` (#6e6e73), `var(--bg-elevated)` (#ffffff) | `var(--text)` (#d4d4d8), `var(--muted)` (#838387), `var(--bg-elevated)` (#191c24) | Primary text, secondary text, status item backgrounds, empty state backgrounds |
| Accent (10%) | `var(--accent)` (#7c5cff) | `var(--accent)` (#d2befc) | Action card hover border, "view all" links, action card icon background, data volume trend line color, ECharts accent |
| Destructive | `var(--destructive)` (light: #7c5cff / mapped from original #dc2626) | `var(--destructive)` (dark: #b08df5) | Critical health status, active alert badge (>0), error text. Note: project uses accent violet for destructive, NOT red. |

Accent reserved for: interactive hover states (action card border/shadow on hover), "view all / see all" navigation links, action card icon container backgrounds, data volume trend line (ECharts series color), status semantic badges (ok/warn/danger backgrounds use `-subtle` variants).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None — dashboard is a read-only overview page. All clickable elements navigate to other pages (navigation, not actions). |
| Empty state — Pie chart (no instances) | "暂无数据库实例" |
| Empty state — Trend chart (no capacity data) | "暂无容量数据，请确保监控采集已启用" |
| Empty state — Health cards (no instances) | "暂无实例健康数据" |
| Empty state — Alerts panel (no unread alerts) | "暂无未处理告警" + subtitle "系统运行正常" |
| Loading state — Full page | "加载中..." (centered, muted text) |
| Error state — Global fetch failure | "{error message}" in `var(--destructive)`. Centered container. |
| Error state — Partial (chart only) | Inline error text in chart container, does not affect other sections. |
| Destructive actions in this phase | None. Dashboard contains no mutation endpoints (no delete, modify, reject, or confirm). All interactions are read/navigate. |

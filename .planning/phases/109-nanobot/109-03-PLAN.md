  <action>
    Add bidirectional checkpoint restore to AgentRunner. Per D-17. The current agent-core has one-way `checkpointCallback` (send only). Add the nanobot restore logic.

    1. **types.ts**: Add checkpoint payload types:
       ```typescript
       export interface RuntimeCheckpoint {
         assistant_message?: Record<string, unknown>;
         completed_tool_results?: Record<string, unknown>[];
         pending_tool_calls?: Record<string, unknown>[];
       }
       ```

    2. **runner.ts**: Add these static/helper methods to or near the AgentRunner class:

       a. `_setRuntimeCheckpoint(session: Session, payload: RuntimeCheckpoint): void`
          - Sets `session.metadata['runtime_checkpoint'] = payload`
          - Does NOT save here (the session manager caller is responsible)
          - Pattern from nanobot loop.py lines 1473-1476

       b. `_clearRuntimeCheckpoint(session: Session): void`
          - Pops `'runtime_checkpoint'` from session.metadata
          - Pattern from nanobot loop.py lines 1484-1486

       c. `_checkpointMessageKey(message: SessionEntry): any[]`
          - Returns tuple of (role, content, tool_call_id, name, tool_calls, reasoning_content, thinking_blocks)
          - Pattern from nanobot loop.py lines 1489-1498

       d. `_restoreRuntimeCheckpoint(session: Session): boolean`
          - Reads `session.metadata['runtime_checkpoint']`
          - If not a valid checkpoint dict, returns false
          - Extracts `assistant_message`, `completed_tool_results`, `pending_tool_calls`
          - Builds restored_messages array:
            - assistant_message (dict, timestamp added)
            - completed_tool_results (each dict, timestamp added)
            - pending_tool_calls → tool role messages with `content: "[Task interrupted before this tool finished.]"`
          - Deduplicates against existing session.messages using `_checkpointMessageKey` overlap detection:
            - Finds max overlap (suffix of existing matches prefix of restored)
            - Appends only non-overlapping restored messages
          - Calls `_clearRuntimeCheckpoint(session)`
          - Returns true if any messages were restored
          - Pattern from nanobot loop.py lines 1500-1552

       e. **Integration into run()**: The checkpoint restore is NOT called inside AgentRunner itself. It's called by the DirectAdapter (or any caller) BEFORE calling `run()`. The existing checkpointCallback (line 796-797) is already called by the runner during tool execution — that stays one-way (send). The RESTORE is caller responsibility.

       f. Add `_restoreRuntimeCheckpointForMessages(session: Session): Message[]`:
          - Calls _restoreRuntimeCheckpoint
          - Returns session.messages as Message[] (from SessionEntry[])
          - Convenience for callers that need to feed restored messages into AgentRunner.run()

    3. **Test runner-checkpoint.test.ts**: Create test:
       - Create Session with some messages
       - Manually set a checkpoint in session.metadata with known payload
       - Call _restoreRuntimeCheckpoint — verify restored messages are appended
       - Set checkpoint with messages that overlap existing history — verify dedup works (no duplicates)
       - Set checkpoint with pending_tool_calls — verify backfill with "[Task interrupted]" content
       - Clear checkpoint after restore — verify metadata no longer has checkpoint key

    4. **index.ts**: Export `RuntimeCheckpoint` type if needed externally.

    IMPORTANT: The checkpoint mechanism is caller-side responsibility (DirectAdapter calls restore before run, the runner's own checkpointCallback sends during execution, and runner's finalize clears). Do NOT modify the AgentRunner.run() loop itself for restore — only add the static helpers.

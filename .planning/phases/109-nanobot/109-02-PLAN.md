    1. **context.ts**: Create `ContextBuilder` class:

       - Constructor takes `workspace: string`, optional `timezone?: string`, optional `disabledSkills?: Set<string>`
       - Creates internal `MemoryStore` and `SkillsLoader` instances using the workspace path

       - `buildSystemPrompt(skillNames?: string[], sessionSummary?: string): string`
         Assembles system prompt in this order (from nanobot context.py):

         1. **Identity section**: Reads SOUL.md for agent identity/role. Falls back to generic "database operations assistant" if SOUL.md not found. (Per research Pitfall 2.)
         2. **Bootstrap files**: Loads AGENTS.md and HEARTBEAT.md content. Join with `\n\n---\n\n`.
         3. **Memory context**: Calls `memoryStore.getMemoryContext()`. Includes MEMORY.md content and recent history summary.
         4. **Skills**: If skillNames provided, calls `skillsLoader.loadSkillsForContext(skillNames)`. Otherwise calls `skillsLoader.buildSkillsSummary()`.
         5. **Session summary**: If provided, appends last conversation summary.

         Join all parts with `\n\n---\n\n`.

       - `buildMessages(history: SessionEntry[], currentMessage: string, skillNames?: string[]): Message[]`
         1. Calls `buildSystemPrompt(skillNames)` for the system message
         2. Builds runtime context lines (time, timezone, workspace info) via private `_buildRuntimeContext()`
         3. Appends runtime context to currentMessage: `currentMessage + '\n\n' + runtimeContext`
         4. Returns `[{ role: 'system', content: systemPrompt }, ...history, { role: 'user', content: mergedContent }]`

       - Private helpers:
         - `_buildRuntimeContext(): string` — builds runtime info: current time, timezone, channel type
         - `_loadBootstrapFiles(): string` — loads SOUL.md, AGENTS.md, HEARTBEAT.md from workspace, returns joined content. Handle missing files gracefully (return empty section).
         - `_getIdentity(): string` — reads SOUL.md for identity, returns description

       - **Bootstrap file list**: `['AGENTS.md', 'SOUL.md', 'HEARTBEAT.md']` (per D-11). These are read from the workspace root. Handle missing files with empty string (no crash).

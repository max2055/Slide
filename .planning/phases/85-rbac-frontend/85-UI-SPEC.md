# Phase 85 — RBAC Frontend — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Context sources:** CONTEXT.md (decisions D-01 through D-13), base.css (design tokens), existing views (llm-config.ts, users-management.ts).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Lit Web Components) |
| Preset | not applicable |
| Component library | none (custom LitElement + CSS custom properties) |
| Icon library | OpenClaw native SVG icons (`frontend/src/openclaw/ui/icons.ts`) |
| Font | Inter (body), JetBrains Mono (mono) — defined in `--font-body` / `--mono` |

The existing design system is fully tokenized via CSS custom properties in `base.css`. All new components MUST use these tokens (e.g. `var(--card)`, `var(--border)`, `var(--accent)`, `var(--bg-elevated)`, `var(--text)`, `var(--muted)`, `var(--radius-*)`, `var(--shadow-*)`). Do NOT introduce new hardcoded color/radius/typography values.

## Spacing Scale

The project uses a ~4px grid with observed values. No new spacing tokens needed — reuse existing CSS variables and common padding/margin values from existing views.

| Token | Observed Value | Usage |
|-------|---------------|-------|
| 4px | 4px | Compact gaps, icon padding |
| 8px | 8px | Tight element spacing, gap in action groups |
| 12px | 12px | Default card padding, table cell padding |
| 16px | 16px | Section padding, modal body padding |
| 20px | 20px | Modal header/footer padding |
| 24px | 24px | Page-level white space |
| 32px | 32px | Major section breaks |
| 48px | 48px | Page-level top margin |

Exceptions: none. The 4px grid is already established across the codebase.

## Color

All values use existing CSS custom properties. No new colors or overrides.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)`, `var(--panel)` | Page background, surface |
| Secondary (30%) | `var(--card)`, `var(--bg-elevated)` | Cards, table rows, dialog backgrounds, sidebar |
| Accent (10%) | `var(--accent)` | **Reserved for:** primary buttons (`btn--primary`), active tab indicators, focus rings (`var(--focus-ring)`), link text, active toggle states |
| Destructive | `var(--destructive)` / `var(--danger)` | Delete buttons, destructive action buttons (hard delete role/permission/user-role), error messages |
| Semantic | `var(--ok)`, `var(--warn)`, `var(--info)` | Status indicators (active/green, warning/yellow, info/blue) |

Accent reserved for: primary action buttons, active sub-tab indicators, focus-visible outlines, interactive element hover highlights, toggle switches in checked state, link text, selected row borders. Accent must NOT be used for backgrounds of non-interactive elements.

**Role badge color mapping** (from users-management.ts, extended for RBAC context):
- `admin` → red (#ef4444)
- `dba` → blue (#3b82f6)
- `developer` → green (#22c55e)
- `analyst` → purple (#a855f7)
- `viewer` → muted (var(--muted))
- `auditor` → amber (#f59e0b)
- Custom roles → use `var(--accent)` with `var(--accent-subtle)` background

### Destructive Actions and Confirmation

| Action | Confirmation Approach |
|--------|----------------------|
| Delete role | `confirm()` dialog: "确定删除角色「{role_name}」？删除后不可恢复。该角色的所有权限关联将被移除。" |
| Delete permission | `confirm()` dialog: "确定删除权限「{permission_code}」？删除后不可恢复。已分配此权限的角色将失去该权限。" |
| Remove role from user | Badge click → no extra confirm (removal is instant, trust user intent). OR soft confirm if requested during implementation. |
| Revoke instance access | Save action in modal (uncheck + save) — user explicitly confirms by clicking Save. No extra confirm needed. |

## States and Behaviors

### List Views (Roles, Permissions)

| State | Behavior |
|-------|----------|
| Loading | Centered text "加载中..." with CSS spinner or skeleton |
| Empty | Centered empty state copy (see Copywriting above) |
| Error | `.error-msg` colored box with error copy |
| Data loaded | Full table with sticky header, row hover, column sorting |
| 403 (no permission) | `.no-permission` message block similar to users-management.ts |

### Modal Dialogs (Create/Edit)

| State | Behavior |
|-------|----------|
| Open | Slide-in animation via `@keyframes modal-slide-in`; overlay backdrop click dismisses |
| Validation error | Inline error messages in `.form-error` style below fields |
| Save in progress | Submit button shows "保存中..." with `?disabled` |
| Save success | Close modal, refresh parent list |
| Save error | `save-error` box at top of modal body |
| Close | Click × button, click overlay, or press Escape |

### Delete Confirmation

| State | Behavior |
|-------|----------|
| Dialog | Native `confirm()` dialog (matching users-management.ts pattern) |
| Delete in progress | Button shows "删除中..." with `?disabled` |
| Delete success | Refresh list |
| Delete error | `.error-msg` at top of page, list remains unchanged |

### User-Role Binding (Split Layout)

| State | Behavior |
|-------|----------|
| No user selected | Left panel shows user list; right panel shows "请选择一个用户" prompt |
| User selected, loading roles | Right panel shows loading spinner |
| User selected, roles loaded | Right panel shows role badges with × remove button + dropdown to add |
| No roles assigned | Right panel shows "暂无角色" with dropdown enabled |
| Add role | Dropdown → select role → call API → refresh list |
| Remove role | Click × on badge → API call → refresh list |

### Permission Checkbox Modal (Edit Permissions per Role)

| State | Behavior |
|-------|----------|
| Loading permissions | Loading spinner |
| Loaded | Permissions grouped by `resource` type using `<details>` accordion pattern |
| All checked/unchecked | Group header shows count (e.g. "已选 5/12") |
| Save | Collect all checked permission IDs → batch API call → close modal |

### Instance Permission Modal

| State | Behavior |
|-------|----------|
| Loading instances | Loading spinner |
| Loaded | Checkbox list of all instances (sorted by name), pre-checked for already granted |
| Save | Compare checked vs granted → call grant/revoke API for each diff → close modal |

### Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| > 900px | 4 sub-tabs in horizontal tab bar; user-role binding uses side-by-side layout |
| <= 900px | Sub-tabs wrap or use dropdown; user-role binding stacks vertically (user list on top, roles below) |
| <= 600px | Tables collapse to card-style layout; modals use full-screen width with smaller padding |

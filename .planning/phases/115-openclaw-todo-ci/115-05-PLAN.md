    Per D-14/D-16, set up the CI infrastructure:

    **Step 1: Create .github/workflows/ directory and ci.yml**

    Create `.github/workflows/ci.yml` with:

    ```yaml
    name: CI

    on:
      pull_request:
        branches: [main]

    jobs:
      backend:
        runs-on: ubuntu-latest
        defaults:
          run:
            working-directory: apps/db-ops-api
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: 22
          - run: npm ci
          - run: npm run typecheck
          - name: Run tests
            run: npx vitest run

      frontend:
        runs-on: ubuntu-latest
        defaults:
          run:
            working-directory: frontend
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: 22
          - run: npm ci
          - name: Lint
            run: npm run lint
          - name: Typecheck
            run: npx tsc --noEmit
          - name: Run tests
            run: npx vitest run
    ```

    - Use `npm ci` (clean install) instead of `npm install` for faster/reproducible CI builds
    - Pin `actions/checkout@v4` and `actions/setup-node@v4` (stable versions)
    - Use `node-version: 22` (matches development environment)

    **Step 2: Add oxlint to root devDependencies**

    In root `package.json`, add `oxlint` to `devDependencies`:

| Boundary | Description |
|----------|-------------|
| GitHub Actions → Repository checkout | Third-party actions (actions/checkout, actions/setup-node) execute during CI — supply chain risk |
| Pull Request → CI trigger | Untrusted PRs from forks could run arbitrary test code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-115-05 | Tampering | CI workflow third-party actions | accept | Using pinned major versions (`@v4`) of standard GitHub Actions. Supply chain attack on these widely-used actions is low-probability. |
| T-115-06 | Information Disclosure | CI test output | accept | Tests run on ubuntu-latest; no secrets are exposed in test steps. No database credentials in CI. |
</threat_model>

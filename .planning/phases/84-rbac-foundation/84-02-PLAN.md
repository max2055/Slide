   ```typescript
   function hasPermission(userPermissions: Set<string>, requiredCode: string): boolean {
     // Direct match first
     if (userPermissions.has(requiredCode)) return true;
     // Super admin wildcard
     if (userPermissions.has('*')) return true;
     // Resource wildcard: resource:*
     const colonIdx = requiredCode.indexOf(':');
     if (colonIdx !== -1) {
       const resourcePrefix = requiredCode.substring(0, colonIdx) + ':*';
       if (userPermissions.has(resourcePrefix)) return true;
     }
     // Action wildcard: *:action
     if (colonIdx !== -1) {
       const actionSuffix = '*:' + requiredCode.substring(colonIdx + 1);
       if (userPermissions.has(actionSuffix)) return true;
     }
     return false;
   }
   ```

4. **Imports:** Import RbacService from `./rbac-service.js`
   ```typescript
   import { RbacService } from './rbac-service.js';
   const rbacService = new RbacService();
   ```

5. **No caching per RESEARCH.md recommendation:** Permission lookups are per-request (no cache). The SQL JOIN with proper indexes is <5ms per request at current scale.

**Endpoints (all protected with requirePermission('admin:*')):**

Roles:
- `GET /roles` — list all roles. Uses RbacService.listRoles()
- `POST /roles` — create role. Body: { name, description }. Uses RbacService.createRole()
- `GET /roles/:id` — get role by ID. Uses RbacService.getRole()
- `PUT /roles/:id` — update role. Body: { name?, description? }. Uses RbacService.updateRole()
- `DELETE /roles/:id` — delete role. Uses RbacService.deleteRole(). Returns 400 if is_system.

Permissions:
- `GET /permissions` — list all permissions. Query param: ?resource=instance. Uses RbacService.listPermissions()
- `POST /permissions` — create permission. Body: { code, name, resource, action, description? }. Uses RbacService.createPermission(). Validate code matches `resource:action` format.
- `DELETE /permissions/:id` — delete permission. Uses RbacService.deletePermission(). Returns 400 if referenced.

Role-Permission assignment:
- `POST /roles/:roleId/permissions` — assign permission to role. Body: { permissionId }. Uses RbacService.assignPermissionToRole()
- `DELETE /roles/:roleId/permissions/:permissionId` — revoke permission from role. Uses RbacService.revokePermissionFromRole()
- `GET /roles/:roleId/permissions` — list role's permissions. Uses RbacService.getRolePermissions()

User-Role assignment:
- `POST /users/:userId/roles` — assign role to user. Body: { roleId }. Uses RbacService.assignRoleToUser()
- `DELETE /users/:userId/roles/:roleId` — revoke role from user. Uses RbacService.revokeRoleFromUser()
- `GET /users/:userId/roles` — list user's roles. Uses RbacService.getUserRoles()

Instance permissions:
- `POST /users/:userId/instances` — grant instance access. Body: { instanceId }. Uses RbacService.grantInstanceAccess()
- `DELETE /users/:userId/instances/:instanceId` — revoke instance access. Uses RbacService.revokeInstanceAccess()
- `GET /users/:userId/instances` — list user's accessible instances. Uses RbacService.getUserInstanceAccess()

|----------|-------------|
| Client -> API | All RBAC management endpoints require admin:* permission. Middleware code factory validates before route handler. |
| API -> RbacService | Middleware calls service methods per-request. No caching ensures permissions are always read from DB. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84-04 | Elevation of Privilege | requirePermission bypass | mitigate | Factory function returns 403 on mismatch. Logic is local scope — no external override possible. |
| T-84-05 | Tampering | instanceId spoofing | mitigate | Canonical rule enforced: instanceId MUST come from request.params.id (not body). Middleware logs warning if body has different instanceId. |
| T-84-06 | Spoofing | Super admin '*' wildcard | mitigate | Only users with '*' in their permission set (currently only admin role) get this. Assignable only via RbacService which requires admin:* permission. |
| T-84-07 | Repudiation | RBAC API operations | accept | Phase 84 is backend only. Phase 85 frontend + logging will be tracked. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Platform code -> getAgentEngine() | Platform calls the factory. The factory returns either DirectAdapter or OpenClawAdapter depending on feature flags. |
| getAgentEngine() -> DirectAdapter or OpenClawAdapter | Factory creates and caches the appropriate adapter instance. Two singletons exist concurrently during dual-run. |
| chat-handler.ts -> IAgentEngine.chat() | Platform RPC handler calls the interface. The underlying adapter runs in-process. |
| ai-agent-bridge.ts -> IAgentEngine.invoke() | Fire-and-forget call. Analysis result is persisted by the tool handler (slide_complete_analysis). |
| server.ts -> engine.start() | server.ts eagerly calls start() after factory returns. The adapter starts its WS transport. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-108-10 | Spoofing | Feature flag env vars | accepted | ENABLE_AGENT_ADAPTER_CHAT and ENABLE_AGENT_ADAPTER_ANALYSIS are read at process start. No runtime toggle supported in Phase 108 (matching D-13's one-time registration principle). Runtime toggle is planned for Phase 110. |
| T-108-11 | Information Disclosure | adapter type in logs | accepted | The adapter type is logged at startup ("[Agent] Engine started: direct"). Log level: info. No sensitive data included. |
| T-108-12 | Elevation of Privilege | DirectAdapter via getAgentEngine() | accepted | Both adapters run in-process with the same Node.js process privileges. The adapter has no access to database credentials or instance connections — those are managed by service instances injected into tool handlers. |
| T-108-13 | Tampering | Dual-run comparison test | accepted | The test creates parallel adapter instances but does not send real LLM calls. Tool schemas are compared with deep equality. No risk of accidental production calls. |

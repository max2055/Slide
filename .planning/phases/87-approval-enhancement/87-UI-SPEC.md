## Spacing Scale

The project uses an 8px-base scale inherited from CSS custom properties. All standard values are multiples of 4.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Tab gap, button padding inner offset, timeline dot gap, checkbox-badge inner gaps, row separators in timeline |
| sm | 8px | Card inner padding, button padding (h/v), note-area padding |
| md | 16px | Card padding, action bar gap, modal body padding |
| lg | 24px | Host padding (`:host { padding: 24px }`), section gaps |
| xl | 32px | Split panel gap (between CodeMirror + info panel) |
| 2xl | 48px | Major section breaks (between timeline and action area) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Row-level execute checkbox column: 44px minimum touch target width (for checkbox alignment)
- Batch action bar: 48px height (dense toolbar for multi-select controls)
- Timeline vertical line: 2px stroke width (connecting line between timeline dots, visually distinct from row separators which are 4px)

Source: Codebase pattern — `approval-dashboard.ts` uses `padding: 24px` on `:host`, `16px` on `.card`, `8px` on `.sql-preview`.

### Accent reserved for (exclusive list)

1. Approve button (primary CTA in both list and batch action bar)
2. Active tab indicator (underline + text color)
3. Focus ring on inputs/buttons/checkboxes
4. "返回列表" link in detail page header
5. CodeMirror gutter markers and cursor
6. Timeline active/highlighted event dot
7. Checkbox checked state
8. "审批后执行" toggle checked state

**Accent is NOT used for:** body copy, muted labels, error messages, secondary buttons, card borders, background fills.

### Destructive reserved for (exclusive list)

1. Reject button (single and batch)
2. "已驳回" status label
3. Rejected timeline event node
4. Batch dismiss action (if user clears all selections)

### Focal Point

The primary visual anchor on the list screen is the batch action bar (`.batch-bar`) with the selected count label and action buttons. When any row is checked, the bar appears between the tabs and card list, drawing immediate user attention via its 48px height, accent-colored approve button, and destructive-colored reject button. This creates a clear visual hierarchy: the selected count communicates scope, and the paired action buttons invite the next step.

Source: `global.css` lines 4-203 (both themes). Existing pattern in `approval-dashboard.ts` lines 20-31.

### Element Breakdown

#### List sub-view (enhanced from existing)

- `.tabs` — Tab bar: "待审批" / "已处理" (unchanged styling)
- `.batch-bar` — New: Action bar that auto-shows/hides (D-04). Contains count label + "通过 (N)" + "驳回 (N)" buttons. Positioned between tabs and card list, height 48px.
- `.card` — Enhanced: Each card now has a checkbox column on the left, and per-row "审批后执行" checkbox for pending items (D-07). In processed tab, cards show execution result summary inline (D-10).
- `.card-checkbox` — Row-level checkbox, 44px minimum width, left-aligned
- `.card-body` — Content area (unchanged: risk badge, ID, timestamp, SQL preview, AI recommendation)
- `.card-exec-checkbox` — New: "审批后自动执行" row-level checkbox, default checked (opt-out) (D-07). Only visible for pending status items.
- `.card-exec-result` — New: Inline execution result display for processed items (D-10). Shows duration + rows affected for success, error message for failure.
- `.card-status` — Unchanged status indicator layout (existing `r.status === 'executed'` / `rejected` blocks)

#### Detail sub-view (new)

- `.detail-header` — Top bar with "返回列表" link + title "审批详情"
- `.detail-split` — CSS Grid two-column layout (left: CodeMirror, right: metadata + timeline). Gap: 32px.
- `.detail-sql-panel` — Left column. Contains `.cm-wrap` container for CodeMirror 6 read-only editor. Same pattern as `sql-console.ts` but with `EditorView.editable.of(false)` extension (D-03).
- `.detail-sidebar` — Right column. Contains:
  - `.detail-meta-card` — Metadata card: instance name, submitter, submit time, risk level (badge), current status (badge with semantic color)
  - `.timeline-card` — History timeline card (D-02)
- `.timeline-card` — Container with section header "审批历程" + vertical timeline list
- `.timeline-node` — Single event node in timeline. Contains: dot (colored by event type), event name, timestamp, optional detail text. Connected by 2px vertical line on left.
- `.timeline-dot` — 10px circle, color coded: accent for submitted/approved/notified, warn for AI review, ok for auto-execute success, destructive for rejected/execution fail, muted for pending/unresolved.

#### Batch confirmation dialog (new)

- `.modal-overlay` — Fixed overlay (z-index: 1000), dismisses on outside click / Escape (existing pattern from `rbac-page.ts` line 117, `users-management.ts` line 296)
- `.modal` — Card dialog (max-width: 520px). Contains header "批量通过" / "批量驳回", body with count + textarea, footer with cancel + confirm.
- `.modal-body-text` — "确认对选中的 N 项请求执行操作？"
- `.modal-note` — Textarea, same styling as existing `.note-area` but unified for batch (not per-row)
- `.modal-footer-actions` — Cancel button (default) + Confirm button (accent for approve, destructive for reject)

## Interactions

### List mode interactions

| Interaction | Behavior |
|-------------|----------|
| Checkbox toggle | Click toggles selection; checked row gets subtle highlight (`--accent-subtle`). Selecting any row reveals the batch action bar. |
| Select all | Header checkbox (or keyboard shortcut) selects all pending items on current page. |
| Deselect all | Unchecking all items hides the batch action bar (D-04). |
| Batch approve | Click "通过 (N)" opens batch dialog with approve title/button. |
| Batch reject | Click "驳回 (N)" opens batch dialog with reject title/button. |
| Per-row approve | Click inline "通过" opens batch dialog with N=1 (unified modal path). |
| Per-row reject | Click inline "驳回" opens batch dialog with N=1 (unified modal path). |
| Execute checkbox toggle | Per-row "审批后自动执行" checkbox, default checked (D-07). Toggle is independent of row selection. |
| Row click | Clicking anywhere on a card row navigates to detail sub-view for that request. |
| Processed tab row click | Clicking opens detail sub-view in read-only mode showing full history. |
| Tab switch | Clicking "待审批" / "已处理" switches tab and reloads data. Clears any checkbox selections. |

### Detail mode interactions

| Interaction | Behavior |
|-------------|----------|
| Back to list | Click "返回列表" returns to list sub-view, preserving tab state. |
| CodeMirror load | On mount, creates read-only EditorView with `sql({ dialect: MySQL })`, `oneDark`, `basicSetup \ { 'fold' }`, `EditorView.editable.of(false)`. Database type detected from instance info for dialect switching (MySQL/PostgreSQL). |

### Batch dialog interactions

| Interaction | Behavior |
|-------------|----------|
| Open dialog | Confirm button state: disabled until submit completes. |
| Unified note | Single textarea applies to all selected items (D-05). |
| Confirm | Sends POST to backend with array of {id, execute_after_approve} + unified notes. |
| Cancel | Dismisses dialog, preserves selection state. |
| Success | Closes dialog, removes approved/rejected items from list, shows inline success. |
| Error | Shows error message in dialog body, keeps dialog open. Dialog overlay click-to-dismiss preserved. |
| Close on outside click | Clicking overlay backdrop or pressing Escape dismisses dialog with no action. |

### Filtering for batch operations

- When batch action is taken, the system auto-filters selected items to only include `status === 'pending'` items (D-06).
- Non-pending items in the selection are silently excluded, not errored.

## Loading / Empty / Error / Edge Cases

| State | Behavior |
|-------|----------|
| List loading | Existing pattern: `.loading` div with centered muted text (from `sql-console.ts` line 118). |
| List empty (pending) | Centered muted text: "暂无待审批请求" |
| List empty (processed) | Centered muted text: "暂无已处理记录" |
| Detail loading | Centered muted text: "加载中..." inside detail sub-view container. |
| Detail fetch error | Error state card with red border + "加载失败，请重试" + retry button. |
| Batch action network error | Error message displayed inside modal body using existing `.error-box` pattern (red background, monospace text). Dialog does not auto-close on error. |
| Batch partial success | Each item in batch shows independent result (D-10): inline execution summary per-row in processed tab. |
| Batch zero eligible items | If all selected items are non-pending after auto-filter, show inline error "所选项目均已处理，请重新选择" and close dialog. |
| Execute toggle disabled | If backend reports instance is not available, auto-execute checkbox is disabled with tooltip "实例不可用，无法自动执行". |
| Very long SQL | CodeMirror wraps lines (default behavior), word-wrap enabled. SQL preview truncation in list cards: `max-height: 80px; overflow: hidden; text-overflow: ellipsis; white-space: pre-wrap` (existing). |
| Timeline overflow | Vertical timeline scrolls within max-height of detail sidebar. Overflow shows scrollbar. |
| Concurrent approve | Duplicate approval attempt shows error: "该请求已被其他审批人处理". |

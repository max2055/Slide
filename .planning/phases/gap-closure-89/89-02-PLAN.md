<objective>
Prevent diagnoses from being stuck in "running" status indefinitely.

**Purpose:** When the Agent dispatched via `dispatchOrReuse` hangs or the LLM call never completes, the analysis record stays "running" forever, and the frontend polls `/api/ai/analysis/:id/status` indefinitely. This plan adds a timeout mechanism to auto-fail stuck analyses.

**Output:**
- `ai-analysis-database-service.ts` — `checkAndFailStuckAnalyses()` method
</objective>

       Use `pool.query()` (not `execute`) with a raw SQL query:
       ```sql
       SELECT id FROM ai_analysis WHERE status = 'running' AND started_at IS NOT NULL AND started_at < NOW() - INTERVAL 10 MINUTE
       ```

    4. Handle errors gracefully — if pool is null, return `{ failed_count: 0 }`. Log errors but don't throw.

    ```typescript
    async checkAndFailStuckAnalyses(): Promise<{ failed_count: number }>
    ```

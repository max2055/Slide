  <name>Task 1: Add database selector dropdown to SQL Console toolbar</name>
  <files>

  <action>
    **Root cause:** SQL Console only has an instance selector. Users cannot choose which database/schema to query within the instance. The schemas are already loaded from `/api/database/instances/${id}/schema-objects` which returns `[{schema: string, tables: [...]}, ...]`.

    **Fix — Add database dropdown:**

    1. Add a new state variable:
       ```typescript
       @state() private selectedDatabase: string = '';
       ```

    2. Add a database dropdown in the `<div class="toolbar">` block (after the instance selector, around line 956):
       ```html
       <select @change=${(e: Event) => { this.selectedDatabase = (e.target as HTMLSelectElement).value; }}
               ?disabled=${!this.selectedId || this.schemas.length === 0}>
         <option value="">选择数据库...</option>
         ${this.schemas.map(s => html`
           <option value=${s.schema} ?selected=${this.selectedDatabase === s.schema}>${s.schema}</option>
         `)}
       </select>
       ```

    3. Reset `this.selectedDatabase = ''` in `_selectInstance()` (line ~508-521) when a new instance is selected, since schemas change per instance.

    4. Update `_directExecute()` (line ~564) to include the database:
       Change body from `JSON.stringify({ sql })` to `JSON.stringify({ sql, database: this.selectedDatabase || undefined })`.
       (The backend API on Plan 01's updated route expects `database` in the body.)

    5. Update `_submitApproval()` (line ~580) to include the database:
       Change body to `JSON.stringify({ instance_id: this.selectedId, sql_text: sql, database_name: this.selectedDatabase || undefined })`.
       (The backend's updated submit route expects `database_name` in the body — see Plan 01 Task 3.)

    6. Consider a better hint when schemas are empty: The schemas are loaded from `_selectInstance()` but only after the instance is selected. If the user selects an instance and no schema exists, set the dropdown placeholder to "无数据库" or keep "选择数据库..." showing the disabled state.

    7. Style the dropdown to match the existing instance selector styling (same `toolbar select` CSS applies automatically).

    Per D-01: The database dropdown must be populated from the instance's actual schemas, and the selected database must be passed to both execution and approval submission endpoints.
  </action>
  <verify>
    <automated>grep -n 'selectedDatabase\|选择数据库' frontend/src/openclaw/ui/views/sql-console.ts</automated>
  </verify>

    - Dropdown resets when instance selection changes
    - Selected database is included in execute request body as `database`
    - Selected database is included in approval submit body as `database_name`
  </acceptance_criteria>

<objective>
Create Dameng-specific CodeMirror dialect for SQL console

Purpose: Per decision D-01, the SQL console must use a Dameng-specific CodeMirror mode (not Oracle mode). Dameng DM8 has its own system views (V$DM_*), functions (DM_SQL_*), and reserved words that differ from MySQL and Oracle. Using `@codemirror/lang-sql`'s `SQLDialect.define()`, create a custom `DamengDialect` that provides correct syntax highlighting and autocompletion for Dameng SQL dialect. The dialect activates when the user selects a Dameng database instance in the SQL console.

Output:
- `DamengDialect` defined via `SQLDialect.define()` with Dameng-specific keywords, builtins, and types
- Dialect switching logic in sql-console.ts that selects DamengDialect when instance db_type === 'dameng'
- Autocompletion integration with Dameng system view names
</objective>

### 1. Update import (line 6)

```

```
To:

```
import { sql, MySQL, SQLDialect } from "@codemirror/lang-sql";
```
Keep `MySQL` unchanged (still needed for MySQL instances). Add `SQLDialect` for defining the custom dialect.

### 2. Define DamengDialect as a module-level constant

After the imports block (before the `@customElement("sql-console-page")` class declaration), add:

```
// Dameng DM8 SQL dialect for CodeMirror
const DamengDialect = SQLDialect.define({
  ...
});
```

Populate the dialect with the full keyword set from RESEARCH.md Pattern 2. The key Dameng-specific identifiers beyond standard SQL/PLSQL:

- **builtin** must include: V$BUFFER_POOL_STATISTICS V$DATAFILE V$DEADLOCK_HISTORY V$INSTANCE V$LOCK V$MEMORY_INFO V$PARAMETER V$SESSION V$SESSIONS V$SQLAREA V$SQL_HISTORY V$SYSSTAT V$VERSION DBA_DATA_FILES DBA_SEGMENTS DBA_TABLES DBA_TABLESPACES ALL_TABLES ALL_TAB_COLUMNS USER_TABLES DM_INI DBMS_XPLAN PLAN_TABLE DM_SQL_CREATE_HUGE_TABLE TO_CHAR TO_DATE TO_NUMBER NVL DUAL

- **keywords**: Use the full list from RESEARCH.md (the large space-separated block of SQL keywords including DM, TABLESPACE, TRIGGER, SYNONYM, SEQUENCE, etc.)

- **types**: BIGINT BINARY BIT BLOB BOOLEAN CHAR CLOB DATE DATETIME DEC DECIMAL DOUBLE FLOAT IDENTITY INT INTEGER INTERVAL MEDIUMINT NATIONAL NCHAR NUMBER NUMERIC NVARCHAR RAW REAL ROWID SERIAL SMALLINT TEXT TIME TIMESTAMP TINYINT VARCHAR VARCHAR2 VARRAY VARYING XML

- **Configuration flags**: 
  - `plsqlQuotingMechanism: true` (Dameng is Oracle-compatible)
  - `doubleQuotedStrings: false` (double quotes are identifiers)
  - `spaceAfterDashes: false`
  - `identifierQuotes: '"[]'`
  - `caseInsensitiveIdentifiers: true` (CRITICAL per Pitfall 4)

### 3. Add dialect selection logic at editor creation point

Find where `EditorState.create()` is called — the current code uses `sql({ dialect: MySQL, ... })`. Replace the hardcoded MySQL dialect with a dynamic selection:

```
let selectedDialect = MySQL;
if (this.selectedInstance?.db_type === 'dameng') {
  selectedDialect = DamengDialect;
}
const lang = sql({ dialect: selectedDialect, upperCaseKeywords: selectedDialect === DamengDialect });
```

The key constraint: `selectedDialect === DamengDialect` evaluates to true only when the dialect is the Dameng one, enabling `upperCaseKeywords: true` for Dameng (since Dameng keywords are traditionally uppercase).

### 4. Integrate into instance change flow

If the editor is recreated when the user selects a different instance (look for instance selection handlers, likely around line ~500 where `loadSchemaObjects` is called), the dialect logic from step 3 must be applied there too. Search for patterns like `editorView`, `EditorState.create`, or `reloadEditor` in the file.

The invariant: every time the editor is created or recreated (whether on firstUpdated, on instance change, or on new tab creation), it must check the current instance's db_type to pick the correct dialect.

Do NOT modify:
- The autocompletion source `sqlCompletionSource` — CodeMirror's built-in SQL completion already adapts to the dialect
- The SQL execution logic (executeSql API calls)
- The explain plan logic
- The result display rendering
- The schema browser
- Any CSS styles
- Any HTML templates

**现有架构理解：**
- `DatabaseService.connections: Map<number, DatabaseConnection>` 管理所有纳管数据库的连接
- `connected` 字段在 `addConnection` 成功时置为 `true`，但**从未在任何地方置为 `false`**
- 所有公开方法（`getRealtimeMetrics`、`getSlowQueries`、`getSchemaObjects`、`getQueryAnalytics`、`getExplainPlan`、`getActiveSessions`、`getCapacityInfo`）入口都检查 `!conn || !conn.connected` 并返回 `null`。但 `connected` 字段不会自动更新
- `checkHealth` 方法已存在，但它执行的是"数据库实例的健康检查"（连接数使用率、缓存命中率等），不专门检测连接是否实际存活
- `SqlExecutor.executeSql` 直接通过 `databaseService.getConnection(instanceId)` 获取连接后再访问 `conn.pool.query()` / `conn.pgClient.query()` / `conn.oracleConnection.execute()`，不做额外健康检测

**三种断开场景处理策略：**
1. **连接的 DB 整个宕机后重启** — mysql2 pool 自动重连但不可靠，增加 keepAlive + 强制检测
2. **单次查询时连接断开** — 查询异常抛出，catch 后尝试重连
3. **两次查询之间连接断开** — 下次查询前 health check 检测到死连接，重连后执行

    ```
    get conn = this.connections.get(id)
    if (!conn) return false

    try:
      if db_type === 'mysql' && conn.pool:
        const [connection] = await conn.pool.getConnection()  // 从池拿一条连接
        await connection.ping()                                // 实际 PING
        connection.release()
      else if db_type === 'postgresql' && conn.pgClient:
        await conn.pgClient.query('SELECT 1')
      else if db_type === 'oracle' && conn.oracleConnection:
        await conn.oracleConnection.execute('SELECT 1 FROM DUAL')
      else if db_type === 'dameng' && conn.dmConnection:
        await conn.dmConnection.execute('SELECT 1 FROM DUAL')
      else:
        return false  // 不支持的 db_type 或连接不存在

      conn.connected = true   // 确保标记正确
      return true

    catch (error):
      conn.connected = false  // 查询失败说明连接真死了
      console.error(`[checkConnectionAlive] 实例 ${id} (${conn.name}) 连接不可用:`, error)
      return false
    ```

    **4. 新增 `async _withAutoReconnect<T>(id: number, fn: (conn: DatabaseConnection) => Promise<T | null>, methodName: string): Promise<T | null>` 私有方法**
    功能：所有公开查询方法的统一入口包装。自动处理"连接已死→重连→重试"周期。

    逻辑：
    ```
    let conn = this.connections.get(id)
    if (!conn) return null

    // 第一步：如果 connected 标记为 false，尝试一次重连
    if (!conn.connected):
      console.log(`[_withAutoReconnect] 实例 ${id} connected=false，尝试重连`)
      await this.reconnect(id)
      conn = this.connections.get(id)
      if (!conn || !conn.connected) return null  // 重连失败

    // 第二步：执行实际查询
    try:
      return await fn(conn)
    catch (error):
      // 判断是否为连接相关错误
      const isConnError =
        (error instanceof Error && (
          error.message.includes('ECONNREFUSED') ||
          error.message.includes('ETIMEDOUT') ||
          error.message.includes('EHOSTUNREACH') ||
          error.message.includes('PROTOCOL_CONNECTION_LOST') ||
          error.message.includes('read ECONNRESET') ||
          error.message.includes('Cannot enqueue') ||
          error.message.includes('Connection lost') ||
          error.message.includes('closed connection') ||
          error.message.includes('ORA-03113') ||     // Oracle: end-of-file on communication channel
          error.message.includes('ORA-03135') ||     // Oracle: connection lost contact
          error.message.includes('ORA-00028') ||     // Oracle: your session has been killed
          error.message.includes('socket') ||         // 通用 socket 错误
          error.message.includes('not connected')     // pg: no connection
        ))

      if (!isConnError):
        // 不是连接错误，如实上抛（或返回 null）
        console.error(`[${methodName}] 实例 ${id} 查询失败（非连接错误）:`, error)
        return null

      // 是连接错误：标记 disconnected，重连，重试一次
      console.error(`[_withAutoReconnect] 实例 ${id} 连接错误，尝试重连后重试:`, error)
      conn.connected = false
      await this.reconnect(id)
      conn = this.connections.get(id)
      if (!conn || !conn.connected) return null  // 重连失败

      try:
        return await fn(conn)
      catch (retryError):
        console.error(`[${methodName}] 实例 ${id} 重连后重试仍失败:`, retryError)
        return null
    ```

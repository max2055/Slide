## 背景

monitor-collector.ts 中 checkAlerts() 方法是早期原型遗留——它在采集指标后直接评估阈值并创建告警，绕过 alert-engine 的完整告警生命周期（静默、聚合、升级、通知）。alert-engine 已经具备相同的阈值评估能力且为主入口，所以 checkAlerts() 可安全移除。

health_score: 100 出现在 3 个位置：
- report-service.ts L321：collectHealthMetrics() 的返回对象 —— 默认永远返回 100
- monitor-collector.ts L200：重连成功路径，写死 100 和 'healthy'
- monitor-collector.ts L228：异常后重连成功路径，同上

databaseService.checkHealth()（database-service.ts L1399）已正确实现各数据库类型的健康评分计算，现被 updateHealthStatusFromCheck()（monitor-collector.ts L274-307）使用。直接复用即可。

```typescript
  try {
    const health = await databaseService.checkHealth(instanceId);
    if (health) {
      await instanceDatabaseService.updateHealthStatus(
        instanceId, health.health_score, health.status
      );
      await instanceDatabaseService.recordHealthCheck(instanceId, health);
    } else {
      await instanceDatabaseService.updateHealthStatus(instanceId, 0, 'critical');
    }
  } catch (error) { ... }
}
```

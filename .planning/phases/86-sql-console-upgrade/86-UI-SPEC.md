## Spacing Scale

Declared values (multiples of 4) — sourced from existing usage in `sql-console.ts` and project-wide patterns in `base.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline element gaps, icon padding, compact spacing |
| sm | 8px | Button padding (Y), card padding, compact element spacing |
| md | 16px | Default element spacing, card padding (X), section gaps |
| lg | 24px | Section padding, panel gaps, tree node indent |
| xl | 32px | Layout gaps, modal padding |
| 2xl | 48px | Major section breaks, empty-state padding |
| 3xl | 64px | Page-level spacing (top-level container padding) |

**Exceptions:**
- CodeMirror editor uses 13px monospace font — no spacing adjustment needed beyond `--mono` font-family
- Existing `sql-console.ts` uses 6px/10px/12px/14px for fine-tuning: these are non-standard gap values for specific compact controls (button padding, select padding, toolbar gaps). Maintain existing values for backward compatibility; use sm/md standard tokens for new elements.
- Touch targets for icon-only buttons: minimum 28x28px (`sql-console.ts` existing pattern)
- Column headers (sortable): maintain existing 6px padding X / 10px padding Y (`sql-console.ts` line 49)

**Source:** `sql-console.ts` lines 17-55 (existing layout values) and `base.css` lines 458-475 (global spacing).

## Copywriting Contract

All copy follows the existing Chinese UI convention established in `sql-console.ts`. English terms for code/SQL keywords remain untranslated.

### Primary CTAs

| Element | Copy | Notes |
|---------|------|-------|
| Execute query | 执行 (Ctrl+Enter) | Existing, unchanged |
| EXPLAIN plan | 分析计划 | New. Sits next to "执行" in toolbar |
| New tab | + | Icon-only button in tab bar |
| Export CSV | 导出 CSV | Button in results header |
| Switch to Schema view | 结构 | Toggle label in sidebar tab |
| Switch to History view | 历史 | Toggle label in sidebar tab |

### Empty State Copy

| Component | Heading | Body |
|-----------|---------|------|
| Query history (no data) | 暂无查询记录 | 执行查询后，历史记录将自动保存于此 |
| Schema browser (no instance) | — | 请先选择数据库实例以加载 Schema |
| Results panel (no query) | 查询结果 | 执行查询后，结果将显示在此处 |
| EXPLAIN panel (no data) | — | 请先执行查询，然后点击"分析计划"查看执行计划 |
| CSV export (no results) | — | 无数据可导出 |

### Error State Copy

| Context | Problem | Solution Path |
|---------|---------|---------------|
| Query execution | 查询执行失败: {error_message} | 检查 SQL 语法后重试，或联系管理员 |
| Schema load failure | 加载 Schema 失败 | 请检查数据库实例连接状态后重试 |
| History load failure | 加载查询历史失败 | 请刷新页面重试 |
| CSV export failure | 导出 CSV 失败 | 重试或检查浏览器下载权限 |
| EXPLAIN preview failure | 获取执行计划失败 | {error_message} — 请检查 SQL 是否为 SELECT/WITH/SHOW/DESCRIBE 类型 |

### Destructive Confirmation

| Action | Confirmation Copy | Implementation |
|--------|-------------------|----------------|
| Close tab with unsaved SQL | 关闭标签页后将丢失未保存的 SQL 内容，确认关闭？ | Confirmation dialog (D-03) with "确认关闭" and "取消" buttons |
| Close tab (empty/unsaved) | — | Close immediately, no dialog |
| Clear query history | 确定清空查询历史？此操作不可撤销。 | Confirmation dialog with "确认清空" and "取消" buttons |

### Status Messages

## Interaction Contracts

### Visual Hierarchy

| Element | Priority | Description |
|---------|----------|-------------|
| Multi-tab CodeMirror editor area | **Primary focal point** | The active tab's editor occupies the top-center of the main panel. Execute and EXPLAIN CTA buttons sit directly above it in the toolbar. This is the dominant visual element the user interacts with most. |
| Results data table / EXPLAIN visualization | **Secondary** | Occupies the lower portion of the main panel below the editor. Receives results after execution. |
| Left sidebar (Schema/History) | **Tertiary** | Browsable but not the primary focus. Collapsible or toggled. |
| Tab bar | **Tertiary** | Horizontal tab strip above the toolbar. Compact, always visible but not dominant. |

### Multi-Tab Editor (D-01 through D-04)

| Property | Specification |
|----------|---------------|
| Tab bar layout | Horizontal flex row at top of editor area, above toolba r. Scrollable if tabs exceed width |
| Tab shape | Rounded pill/capsule with label, small close (x) button |
| Tab default name | Generated from first non-empty line of SQL (truncated to 40 chars). If empty: "标签 1", "标签 2", ... |
| Tab rename | Double-click label text -> inline input replaces label. Press Enter or blur to confirm. Press Escape to cancel |
| Tab close | Click x button. If editor content non-empty: show confirmation dialog (D-03). If empty: close immediately |
| Tab count limit | Soft cap at 10. When >=10 open, show inline warning in tab bar |
| Active tab indicator | Underline or bottom-border in accent color. Bold text |
| Tab persistence | localStorage key: `sql-console-tabs`. Structure: `{ tabs: [{ id, name, sql }], activeId, instanceId }`. Restored on page load. Result sets NOT persisted (D-02) |
| Shared instance | All tabs share same selected instance. Changing instance select updates all tabs, triggers schema reload |

### Result Table Sort (D-05, D-07)

| Property | Specification |
|----------|---------------|
| Sort trigger | Click column header |
| Sort cycle | ascending (triangle-up accent) -> descending (triangle-down accent) -> none (muted chevron) |
| Sort indicator | Up/down arrow icon next to column name in sorted header |
| Multi-column sort | Not supported. Sorting one column clears previous sort (D-05) |
| Sort reset | Re-executing query resets to unsorted, page 1 (D-07) |
| Sort implementation | Client-side in-memory array sort. Type coercion for numbers vs strings. null values sort to end regardless of direction |

### Result Table Pagination (D-06, D-07, D-08)

| Property | Specification |
|----------|---------------|
| Page size dropdown | Presets: 25 / 50 / 100 / All. Default: 50 |
| Page info display | "{start}-{end} / {total} 行" format (D-08) |
| Navigation controls | Previous button (<) + page number display + Next button (>) |
| "All" page size | Renders all rows in client memory (no virtual scroll) |
| Pagination reset | Re-executing query resets to page 1 (D-07) |

### CSV Export (D-07 scope, SQLC-04)

| Property | Specification |
|----------|---------------|
| Trigger | "导出 CSV" button in results header bar |
| Implementation | Client-side: `new Blob([csvString])` + `URL.createObjectURL()` + programmatic click on hidden download link |
| Filename | `query-result-{instance_name}-{YYYYMMDD-HHmmss}.csv` |
| CSV escaping | Cells containing comma, double-quote, or newline wrapped in double-quotes. Existing double-quotes escaped as "" |
| Header row | Column names as first row |
| Null handling | NULL cells exported as empty string |
| Encoding | UTF-8 with BOM for Excel compatibility |

### Query History Panel (D-09 through D-12)

| Property | Specification |
|----------|---------------|
| Panel location | Left sidebar, same position as schema browser |
| Toggle mechanism | Tab bar within sidebar: "结构" (Schema) / "历史" (History) tabs |
| History list | Vertical scrollable list. Each item shows: SQL preview (single line, truncated 80 chars), instance name, duration, row count, timestamp |
| History search | Text input at top of history panel. Full-text search against stored SQL text |
| Instance filter | Dropdown to filter by database instance |
| Click behavior | Click history item -> SQL replaces current active tab's content. Tab name does NOT change |
| Scroll loading | Last 200 items. Load next batch on scroll-to-bottom |
| History API | New endpoint `GET /api/instances/:id/query-history?limit=50&offset=...&search=...&instance_id=...` (backed by audit_log table) |
| History icon | Use `icons.history` for panel header icon |

### EXPLAIN Visualization (D-13 through D-16)

| Property | Specification |
|----------|---------------|
| Trigger | "分析计划" button next to Execute button |
| Constraint | Only sends for SELECT/WITH/SHOW/DESCRIBE queries (back end enforces, frontend grays out button for other types) |
| View location | Inline in results area, replaces data table |
| View toggle | Button group: "树形" (Tree) / "表格" (Table). Tree is default |
| Tree view | Recursive Lit template for nested JSON plan nodes. Each node shows: operation name, rows estimate, cost, additional details. Collapsible child nodes |
| Table view | Flat sortable table with columns: 操作 (Operation), 行数 (Rows), 成本 (Cost), 详情 (Detail) |
| Summary bar | Above visualization: "总成本: {cost} | 扫描行数: {rows} | {efficiency_grade}" |
| Efficiency grades | "索引扫描 — 效率高" (Index Scan), "全表扫描 — 需检查索引" (Full Table Scan), "临时表 — 需优化" (Temporary table), "文件排序 — 可优化" (Filesort) |
| Back link | "返回结果" link to return to data table view |
| Normalizer | Client-side normalizer function to map MySQL FORMAT=JSON and PostgreSQL EXPLAIN JSON to common plan node format |
| EXPLAIN icon | Use `icons.eye` for EXPLAIN button |

### History/EXPLAIN Tree Component

| Property | Specification |
|----------|---------------|
| Tree node structure | Flex row: expand/collapse chevron (icons.chevronRight / icons.chevronDown) + operation icon + operation label + cost badge |
| Expand/collapse | Click chevron or row to toggle children. Animated slide |
| Node indentation | 16px per nesting level |
| Cost badge | Small rounded pill, right-aligned. Color: var(--muted) |

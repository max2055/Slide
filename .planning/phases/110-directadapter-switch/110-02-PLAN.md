    Enhance DirectGatewayClient with two capabilities in direct-gateway.ts:

    **A. Auth frame on WS open (D-09):**
    In the `onopen` handler, after `this.reconnectAttempts = 0; this.onStateChange('connected');`, add:
    ```typescript
    const token = typeof window !== 'undefined'
      ? (window as any).__apiClient?.getToken?.()
      : null;
    if (token) {
      this.ws!.send(JSON.stringify({ type: 'auth', token }));
    }
    ```
    This sends JWT as the first message after WS connection. The token must be fetched at connect time (not constructor time) to handle transparent refresh (Pitfall 2 in RESEARCH.md).

    **B. 4001 close code detection (D-10):**
    In the `onclose` handler, change from:
    ```typescript
    this.ws.onclose = () => {
      this.ws = null;
      this.onStateChange('disconnected');
      if (!this.closed) {
        this.scheduleReconnect();
      }
    };
    ```
    To:
    ```typescript
    this.ws.onclose = (ev: CloseEvent) => {
      this.ws = null;
      if (ev.code === 4001) {
        this.onStateChange('auth_failed');
        return; // Don't schedule reconnect
      }
      this.onStateChange('disconnected');
      if (!this.closed) {
        this.scheduleReconnect();
      }
    };
    ```
    Use `ev.code === 4001` (numeric comparison only, per RESEARCH.md Pitfall 3 — don't depend on ev.reason).

    **C. Expand ConnectionState type:**
    Change the type union from 3 states to 5:
    ```typescript
    export type ConnectionState = 'connecting' | 'connected' | 'disconnected' | 'auth_failed' | 'exhausted';
    ```

    ```typescript
        return;
    ```

    ```typescript
    /** Manual reconnect — resets retry counter. Used after exhausted state. */

      this.closed = false;
      this.reconnectAttempts = 0;
      this.clearReconnectTimer();
      this.connect();
    }
    ```

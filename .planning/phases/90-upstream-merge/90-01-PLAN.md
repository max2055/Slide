    1. Fetch latest upstream: `git fetch upstream`
    2. Ensure clean working tree: verify `git status --porcelain` has no staged/unstaged changes (ignore untracked files)
    3. Run merge: `git merge upstream/main --no-edit`
       - This will auto-merge what it can and stop with conflicts
    4. If merge aborts/fails already, check `git status` for the actual conflict state
    5. Resolve ALL modify/delete conflicts:
       - Per D-02: Slide deleted docs/, Dockerfile, docker-compose.yml → keep deleted
       - For each modify/delete conflict where Slide deleted and upstream modified, run `git rm` on those files
       - Strategy: `git status` shows these as "deleted by us" or "added by them" — use `git rm <file>` to accept Slide's deletion
       - Key patterns to remove: `docs/*`, `Dockerfile`, `docker-compose.yml`, `.codex`, `.detect-secrets.cfg`, `.jscpd.json`, `.mailmap`, `.markdownlint-cli2.jsonc`, `.npmignore`, `.secrets.baseline`, `.shellcheckrc`, `.swiftformat`, `.swiftlint.yml`, `Swabble/`, `.pi/`, `.vscode/`, etc.
       - Approach: find all modify/delete conflicts via `git ls-files -u | awk '$2~/^[23]$/'` or use `git diff --name-only --diff-filter=UD` combined with `git status --porcelain` to find "D " (deleted in HEAD, modified in theirs) entries
       - For each such file: `git rm <file>`
    6. After resolving deletions, verify: `git diff --name-only --diff-filter=U` should show only `.gitignore` and `CHANGELOG.md` (content conflicts)
  </action>
  <verify>

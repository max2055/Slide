<action>
Create a new LitElement component `health-score-tab` at `frontend/src/openclaw/ui/views/health-score-tab.ts`.

This is a self-contained component that renders the full content of the "健康评分" tab. It receives the instanceId as a property and fetches all its own data.

Component structure:

```typescript
import { LitElement, html, css, nothing } from "lit";
import { customElement, property, state } from "lit/decorators.js";
```

**Properties:**
- `@property({ type: Number }) instanceId: number | null = null;`

**State fields:**
- `healthHistory: { health_score: number; created_at: string }[] | null`
- `latestChecks: { name: string; status: string; score: number; message?: string; dimension?: string }[]`
- `healthStatus: string` -- 'healthy' | 'warning' | 'critical' | 'unknown'
- `capabilities: { metricId: string; name: string; available: boolean; lastAttempt?: string }[]`
- `loading: boolean`
- `error: string | null`
- `timeRange: string` -- '24h' | '7d' | '30d' (default '7d')
- `expandedChecks: boolean` -- collapse/expand for per-check detail (default false)

**connectedCallback:** Calls `_loadData()`

**Methods:**
- `_loadData()`: Calls all 3 APIs in parallel via Promise.all:
  1. Fetch `/api/database/instances/${this.instanceId}/health-history?days=7` (will use the timeRange to compute days)
  2. Fetch `/api/database/instances/${this.instanceId}/health-checks`
  3. Fetch `/api/database/instances/${this.instanceId}/collection-capabilities`
- `_onTimeRangeChange(range: string)`: Updates timeRange, re-fetches health-history with appropriate days param (24h=>1, 7d=>7, 30d=>30)

**Render sections (in order):**

Section 1 -- Time range selector (buttons): "24小时" | "7天" | "30天", active button highlighted

Section 2 -- Score trend chart (ECharts via metric-chart component):
- Use `<metric-chart>` component (imported from `../components/metric-chart.js`)
- Build time series from healthHistory data
- Show 4 dimension lines + total line (different colors)
- percentage=true, height="320px"
- If healthHistory is null/empty, show "暂无评分数据" placeholder

Section 3 -- Latest health score card:
- Current total score with color: >=80 green, >=60 yellow, <60 red
- Health status label

Section 4 -- Per-check detail section (collapsible):
- Header: "最近健康检查详情" with expand/collapse toggle button
- When expanded, render each check from latestChecks array
- Each check item shows:
  - Status icon: green checkmark if status==='ok', yellow warning if 'warning', red X if 'critical'
  - Check name (e.g., "连接状态")
  - Score badge (100 green, 70 yellow, 40-0 red)
  - Message text
  - Dimension label (small tag showing which dimension this check belongs to)
- Use CSS `.check-item` class grid/flex layout
- Collapse/expand transition via conditional rendering

Section 5 -- Collection capability badges:
- Header: "采集能力状态"
- Grid of badges, each showing:
  - Metric name (from metric_definitions.name)
  - Green circle icon if available==true, grey circle icon if false
  - Last attempt time if available

**CSS styles (inline via static styles):**
- `.section-title` -- font-weight 600, margin-bottom
- `.time-range-btns` -- flex row, gap
- `.time-range-btn` -- pill button, `.active` variant with accent color
- `.score-card` -- centered big number with status color
- `.check-item` -- flex row, align-items center, padding, border-bottom
- `.check-status-icon` -- 16px icon sized
- `.check-name` -- flex-grow text
- `.check-score` -- badge-style score display (green/yellow/red backgrounds)
- `.check-dimension` -- small muted tag
- `.capability-grid` -- flex wrap, gap
- `.capability-badge` -- small card with icon + name
- `.badge-green` / `.badge-grey` -- circle indicator
- `.collapsible-toggle` -- cursor pointer, user-select none
- `.collapsible-content` -- margin/padding for indentation
- `.loading`, `.error`, `.empty` -- standard status states

Import `icons` from the shared icon module for status icons (check-circle, alert-triangle, x-circle, circle).

Use helper functions:
- `_getScoreColor(score)`: returns CSS color var name based on score
- `_getStatusIcon(status)`: returns icon SVG for ok/warning/critical
- `_getDimensionLabel(dim)`: returns Chinese label for dimension (availability=>可用性, performance=>性能, capacity=>容量, security=>安全性)

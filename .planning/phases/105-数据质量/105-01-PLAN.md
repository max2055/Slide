## health_check_history table schema
```sql
CREATE TABLE health_check_history (
  id BIGINT UNSIGNED AUTO_INCREMENT,
  instance_id INT UNSIGNED NOT NULL,
  health_score INT NOT NULL,
  status ENUM('healthy', 'warning', 'critical') NOT NULL,
  checks JSON NOT NULL,
  issues JSON DEFAULT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  INDEX idx_instance_id (instance_id),
  INDEX idx_created_at (created_at)
);
```

<action>
Create three new service files in apps/db-ops-api/src/:

**scoring-service.ts:**
- Export function `calculateDimensionScores(checks: HealthCheckResult['checks'], dbType: string, weights)`: returns `{ dimensions: Record<string, number>, total: number }`
- Define per-DB-type dimension mapping constant `DIMENSION_MAP`:
  - mysql: '连接状态'→availability, '连接数使用率'→capacity, '慢查询'→performance
  - postgresql: '连接状态'→availability, '连接数使用率'→capacity, '缓存命中率'→performance, '死锁检测'→security
  - oracle: '连接状态'→availability, '连接数使用率'→capacity, '表空间使用率'→capacity, '库缓存命中率'→performance, '死锁检测'→security
  - dameng: '连接状态'→availability, '连接数使用率'→capacity, '缓冲池命中率'→performance, '锁等待检测'→performance, '死锁检测'→security
- Each dimension score = average of all checks mapped to that dimension
- If a dimension has no checks for the DB type, assign neutral 100 (not 0, per RESEARCH.md Pitfall 1)
- Weighted total = sum of (dimensionScore * weight) for all 4 dimensions, Math.round()
- Export `DIMENSION_MAP` constant for test assertions
- Each check in output should get a `dimension` field added: { name, status, score, message, dimension }

**scoring-config-service.ts:**
- Follow ai-analysis-config-service.ts pattern exactly (see interfaces above)
- Config key: `'scoring.weights'`
- DEFAULT_WEIGHTS: { availability: 0.35, performance: 0.35, capacity: 0.20, security: 0.10 }
- Methods: `getWeights(): Promise<Record<string, number>>`, `saveWeights(weights): Promise<{success, error?}>`
- `getWeights()`: read from system_config, merge with defaults (stored values override, defaults fill missing keys)
- `saveWeights()`: validate each value is number 0.0-1.0, validate sum is within ±0.01 of 1.0, return error message on failure
- Reuse `dbConnection.getPool()` same as ai-analysis-config-service

**collection-capabilities.ts:**
- Class `CollectionCapabilityTracker`
- Internal: `Map<number, Map<string, { available: boolean; lastAttempt: number; lastSuccess: number }>>`
- Methods:
  - `recordMetricAttempt(instanceId: number, metricName: string, success: boolean): void` — update in-memory state
  - `getCapabilities(instanceId: number, dbType: string): Array<{ metricId: string; name: string; available: boolean; lastAttempt?: string }>` — merge expected capabilities from metric_definitions (filtered by db_type) with actual success/failure state. Expected-but-not-yet-attempted metrics: available=false (grey). Successfully collected metrics: available=true (green).
- Export singleton `collectionCapabilityTracker`
- Import `metricRegistry` for MetricDefinition list

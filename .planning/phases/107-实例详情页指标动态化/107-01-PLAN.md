    Modify `getHistoricalMetricsWithRange()` (lines 294-374 in metrics-database-service.ts) to accept and use an optional 4th parameter `metricIds?: string[]`.

    Define a `FIXED_COLUMNS` const at the top of the method body (or just above it in the class scope) containing the 11 columns that exist as physical columns in metrics_history and are queried by this function: `['cpu_usage', 'memory_usage', 'disk_usage', 'connections', 'qps', 'tps', 'active_transactions', 'slow_queries', 'buffer_pool_hit_rate', 'threads_running', 'threads_connected']`.

    Backward compat: if `metricIds` is undefined or empty, use these 11 as default (same behavior as today).

    For both the non-aggregated (interval='1m') and aggregated (DATE_FORMAT) paths:

    - Split `activeMetricIds` into `fixedCols` (present in FIXED_COLUMNS) and `dynamicCols` (not in FIXED_COLUMNS).
    - Build SELECT clause:
      - For non-aggregated path (interval='1m'): fixed columns selected directly, dynamic columns via `JSON_EXTRACT(metrics_data, '$.{id}') as \`{id}\``
      - For aggregated path (AVG): `AVG(fixed_col)` for fixed, `AVG(JSON_EXTRACT(metrics_data, '$.{id}')) as \`{id}\`` for dynamic
    - Build result `metrics` Record: iterate `activeMetricIds` instead of `metricKeys`, push `Number(row[id]) || 0` for each.
    - Use template literal for dynamic column identifiers. The column names in JSON_EXTRACT AS clause and the result object keys are the metric ID strings themselves.

    The method signature changes from:
    `async getHistoricalMetricsWithRange(instanceId: number, period: '1h' | '6h' | '24h' | '7d', interval: '1m' | '5m' | '15m' | '1h'): Promise<{ time: string[]; metrics: Record<string, number[]> }>`

    To:
    `async getHistoricalMetricsWithRange(instanceId: number, period: '1h' | '6h' | '24h' | '7d', interval: '1m' | '5m' | '15m' | '1h', metricIds?: string[]): Promise<{ time: string[]; metrics: Record<string, number[]> }>`

    Keep the existing try/catch and error handling identical. Keep `_periodToMs`, `_formatRecordedAt`, and all other methods unchanged. Do NOT modify the backward compat path behavior: when metricIds is undefined, the result is identical to today.

    Specific implementation pattern for aggregated path (the main path since interval is always specified):

    ```typescript
    // Split into fixed vs dynamic
    const activeMetricIds = metricIds && metricIds.length > 0 ? metricIds : [...FIXED_COLUMNS];
    const fixedCols = activeMetricIds.filter(id => FIXED_COLUMNS.includes(id));
    const dynamicCols = activeMetricIds.filter(id => !FIXED_COLUMNS.includes(id));

    // Build SELECT parts
    const selectParts: string[] = [];
    fixedCols.forEach(c => selectParts.push(`AVG(${c}) as \`${c}\``));
    dynamicCols.forEach(c =>
      selectParts.push(`AVG(JSON_EXTRACT(metrics_data, '$.${c}')) as \`${c}\``)
    );

    // ... existing query structure but with selectParts.join(', ') and activeMetricIds for result mapping
    ```

    For the non-aggregated path (interval='1m'), the SELECT parts are column names directly (no AVG()), and dynamic cols use `JSON_EXTRACT(metrics_data, '$.{id}') as \`{id}\`` directly.

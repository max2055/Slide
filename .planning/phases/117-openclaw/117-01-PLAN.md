## 1. 重命名类型定义 + 删除死字段

**文件:** `frontend/src/app/src/config/types.ts`

- 将 `export interface OpenClawConfig` 重命名为 `export interface SlideConfig`
- 删除 `session`、`update`、`bindings` 三个死字段（经搜索确认无 indexed access 使用）
- 保留 `agents` 字段不变
- 在此文件中添加共享 stub 函数（供 Task 3 的修复文件使用）

结果：
```typescript
export interface SlideConfig {
  agents?: {
    defaults?: Record<string, unknown>;
    list?: Array<Record<string, unknown>>;
  };
}

// Stub for former CLI config-file functions (Slide web app does not use config files)
export function loadConfig(): SlideConfig {
  return (typeof window !== "undefined" && (window as any).__SLIDE_CONFIG__) ?? { agents: { defaults: {}, list: [] } };
}
```

## 2. 全局替换 ~97 个文件中的 OpenClawConfig 引用

搜索 `OpenClawConfig` 并替换为 `SlideConfig`，覆盖以下模式：
- `import type { OpenClawConfig }` → `import type { SlideConfig }`
- `cfg: OpenClawConfig` → `cfg: SlideConfig`
- `as OpenClawConfig` → `as SlideConfig`
- `T extends OpenClawConfig` → `T extends SlideConfig`
- `OpenClawConfig["agents"]` → `SlideConfig["agents"]`

使用全局 search-replace（`sed` 或 IDE），~97 个文件一次性替换。纯机械操作，零逻辑变更。

替换后运行 `npx tsc --noEmit` 确认无与此步骤相关的类型错误。

**排除:** `cron-completion-tool.ts`（backend）虽有匹配但实际无引用，无需改动。

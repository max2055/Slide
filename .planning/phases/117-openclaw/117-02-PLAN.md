<objective>
创建后端 Branding 配置服务，提供 GET/PUT API 端点，复用现有 `system_config` 表持久化 4 个 branding 字段。

**Purpose:** 为 Phase 116 创建的 branding 常量提供运行时配置能力。前端 Settings UI 通过此 API 读写 branding 值。
**Output:** `branding-config-service.ts` + 两个 API 路由（GET/PUT `/api/branding/config`）。
</objective>

<tasks>

## 1. 创建 BrandingConfigService

**文件:** `apps/db-ops-api/src/branding-config-service.ts` (NEW)

遵循 `scoring-config-service.ts` 和 `ai-analysis-config-service.ts` 的既有模式：

```typescript
// 默认值 — 与 frontend/src/app/src/branding.ts 保持一致
const DEFAULTS = {
  cli_name: "slide",
  product_name: "Slide",
  env_prefix: "SLIDE",
  state_dir: ".slide",
};

// CONFIG_KEY = 'branding.config'
// 读写方法：
//   getBranding(): Promise<BrandingConfig> — SELECT FROM system_config WHERE config_key = 'branding.config'
//   saveBranding(config): Promise<{success, error?}> — REPLACE INTO system_config
```

**关键设计:**
- `config_value` 存储为 JSON string
- 读取时与 DEFAULTS 合并（DB 值覆盖默认值，默认值填充缺失键）
- DB 不可用时 fallback 到 DEFAULTS
- 验证：所有 4 个字段必须是非空 string（`cli_name` 只允许 `[a-z][a-z0-9-]*`，`env_prefix` 只允许 `[A-Z][A-Z0-9_]*`）

## 2. 注册 API 路由

**文件:** `apps/db-ops-api/server.ts`

```typescript
// GET /api/branding/config
// 返回: { cli_name, product_name, env_prefix, state_dir }

// PUT /api/branding/config
// Body: { cli_name?, product_name?, env_prefix?, state_dir? }  (部分更新)
// 返回: { success: true } 或 { success: false, error: "..." }
```

路由注册在 scoring config 路由附近，复用 `auth` middleware（仅已认证用户可修改）。

## 3. 验证

```bash
# 获取 token（先登录）
TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Tpam1234"}' | jq -r '.token')

# 测试 GET
curl http://localhost:3000/api/branding/config
# → {"cli_name":"slide","product_name":"Slide","env_prefix":"SLIDE","state_dir":".slide"}

# 测试 PUT
curl -X PUT http://localhost:3000/api/branding/config \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"product_name":"MyApp"}'
# → {"success":true}

# 确认修改已持久化
curl http://localhost:3000/api/branding/config
# → product_name 应为 "MyApp"
```

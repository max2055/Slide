## 1. branding.ts 重构为 Getter + 缓存模式

**文件:** `frontend/src/app/src/branding.ts`

当前状态：纯常量模块，所有值在 import 时快照。

目标架构：
```typescript
// 编译时默认值（DB 不可用时 fallback）
const DEFAULTS = {
  CLI_NAME: "slide",
  PRODUCT_NAME: "Slide",
  ENV_PREFIX: "SLIDE",
  STATE_DIR: ".slide",
} as const;

// 内存缓存（运行时从 DB 加载的值覆盖默认值）
let _cache: BrandingOverrides | null = null;

// 内部：从 API 加载 branding 配置
async function _loadFromApi(): Promise<BrandingOverrides> { ... }

// 公共 getter（所有现有 import 点改用这些）
export function getCliName(): string { ... }
export function getProductName(): string { ... }
export function getEnvPrefix(): string { ... }
export function getStateDir(): string { ... }
export function buildEnvVar(suffix: string): string { ... }
export function buildSymbolKey(key: string): string { ... }
export function buildCliCmd(sub?: string): string { ... }
export function buildUserAgent(): string { ... }

// 供 branding-settings 使用
export async function refreshBrandingCache(): Promise<void> { ... }
export function getBrandingSnapshot(): BrandingConfig { ... }
```

**关键设计决策:**
- 向后兼容：`buildEnvVar`/`buildSymbolKey`/`buildCliCmd`/`buildUserAgent` 函数签名保持不变，内部从 getter 获取值
- 初始化：首次 import 时异步加载 API 值（`_loadFromApi()`），加载完成前返回 DEFAULTS
- 缓存刷新：`refreshBrandingCache()` 在 branding-settings 保存成功后调用
- 4 个原始常量（`CLI_NAME`/`PRODUCT_NAME`/`ENV_PREFIX`/`STATE_DIR`）保留为 `get*()` 函数的 fallback 值，但不再作为模块级 export

**适应现有 import 点:**
- `import { CLI_NAME } from '../branding.js'` → `import { getCliName } from '../branding.js'` + 调用处改为 `getCliName()`
- `import { buildEnvVar } from '../branding.js'` → 保持不变（函数签名不变）
- ~26 个文件，约 30 处 import 需要适配

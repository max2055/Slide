```typescript
const [groups] = await pool.execute(
  `SELECT instance_id, alert_type, metric_name,
          FLOOR(UNIX_TIMESTAMP(created_at) / 300) * 300 AS time_bucket,
          GROUP_CONCAT(id ORDER BY id) AS alert_ids,
          COUNT(*) AS cnt,
          MAX(FIELD(level, 'info', 'warning', 'error', 'critical', 'p0')) AS max_level
   FROM alerts
   WHERE status IN ('unread', 'read')
     AND created_at >= DATE_SUB(NOW(), INTERVAL 10 MINUTE)
     AND NOT EXISTS (SELECT 1 FROM alert_event_members m WHERE m.alert_id = alerts.id)
   GROUP BY instance_id, alert_type, metric_name, time_bucket
   HAVING cnt >= 2`
);
```

**Current behavior**: Groups alerts by `FLOOR(UNIX_TIMESTAMP / 300) * 300`. This creates 5-minute buckets at fixed boundaries (e.g., 0:00-4:59, 5:00-9:59). An alert at 4:58 and another at 5:02 -- both for the same issue -- end up in different events.

**Target behavior**: For each unaggregated alert, find any existing open event for the same `instance_id + alert_type + metric_name` where the latest member alert's `created_at` is within 10 minutes. If found, group into that event. If not, start a new event.

The remaining code in aggregate() (event creation, member association, RCA trigger) stays the same.

# Phase 112 — UI Design Contract

> Visual and interaction contract for Phase 112. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Scope Summary

Two parallel tracks:

| Track | UI Impact |
|-------|-----------|
| A. Frontend cleanup | Zero new UI. Delete dead code, rename `openclaw/` to `app/`, remove placeholder views, clean navigation/i18n. |
| B. Cron tasks management | **New UI** — Settings page "cron-jobs" tab with job list, enable/disable, cron expression editor, manual trigger, run log viewer. |

This contract covers Track B only (the new UI). Track A has no visual or interaction contract — it is pure deletion/rename.

## Interaction Contracts

### 1. Settings Tab Integration

- Navigation path: `/cron-jobs` (new path, under "settings" group)
- Tab title (i18n key): `tabs.cron-jobs` — Chinese: "定时任务"
- Tab icon: `loader` (same as existing cron tab icon, icon already exists in icons.ts)
- Permission required: `admin:*` or `dba:*` (per D-20)
- Tab appears as a new entry in the "settings" group alongside "llm-config", "ai-settings", "scoring-settings"
- The old top-level `/cron` tab and its navigation entry are **removed** (Track A)

### 2. Cron Job List Table

- Renders on tab load. Loading state: full-width "loading..." text centered, matching `ai-settings.ts` pattern.
- Columns:
  | Column | Width | Content |
  |--------|-------|---------|
  | Status | 48px | Green/gray dot indicator (enabled = green dot `--ok`, disabled = gray dot `--muted`) |
  | Name | 1fr | Job name, semibold, `--text-sm` |
  | Description | 2fr | Human-readable description, `--text-sm`, `--muted` color |
  | Cron Expression | 180px | Monospace, `--text-xs` |
  | Next Run | 140px | Relative time ("in 5 min"), `--text-xs`, `--muted` |
  | Last Run | 140px | Relative time ("2 min ago"), `--text-xs` |
  | Last Result | 80px | Status badge: OK / Error / Skipped / N/A, colored dot + text |
  | Actions | 120px | Toggle switch + "Trigger" btn + "Logs" btn |

- Row hover: `--bg-hover` background highlight.
- Alternating row backgrounds: not used (keep flat `--card`).
- Empty state: should never occur (13 seeded jobs exist). If API returns 0 jobs, show: "No cron jobs configured" with a muted icon.

### 3. Enable/Disable Toggle

- Reuses `.cfg-toggle` pattern from `ai-settings.ts` (40px x 22px slider)
- Click toggles immediately with optimistic UI update
- On failure: revert toggle + show error toast ("Failed to update cron job status")
- No confirmation dialog needed for toggle (low-impact, immediately reversible)

### 4. Cron Expression Edit

- **Trigger:** Double-click on the cron expression cell or click an edit icon
- **Mode:** Inline edit or slide-up panel (prefer inline edit for simplicity)
- **Inline edit controls:**
  - Text input (`--mono` font, `--text-xs`, auto-width matching content)
  - "Save" icon button (checkmark) — saves via PATCH
  - "Cancel" icon button (X) — reverts to original value
  - Error state: input border turns `--danger` if expression is invalid
- **Below the input** (or in a popover): **Cron expression preview** — show next 5 execution times as a compact list:
  ```
  2026-05-27 14:00
  2026-05-27 15:00
  2026-05-27 16:00
  2026-05-27 17:00
  2026-05-27 18:00
  ```
- **Common template dropdown:** a select/dropdown next to the input with presets:
  | Label | Cron Expression |
  |-------|----------------|
  | Every minute | `* * * * *` |
  | Every 5 minutes | `*/5 * * * *` |
  | Every hour | `0 * * * *` |
  | Every day (midnight) | `0 0 * * *` |
  | Every day (08:00) | `0 8 * * *` |
- Selecting a preset populates the input field and triggers preview update.

### 5. Manual Trigger

- **Button:** "Trigger Now" ghost button in the Actions column
- **Click behavior:** Opens a confirmation dialog:

  ```
  ┌──────────────────────────────────────┐
  │ Manual Trigger                       │
  │                                      │
  │ Are you sure you want to manually    │
  │ trigger "{job_name}" now?            │
  │                                      │
  │         [Cancel]  [Confirm Trigger]  │
  └──────────────────────────────────────┘
  ```

- On confirm:
  1. Button enters loading state (disable + spinner)
  2. POST sent to `/api/cron-jobs/:id/trigger`
  3. On success: close dialog, update last run column with "triggered" status, show toast "Triggered successfully"
  4. On failure: keep dialog open, show error "Failed to trigger job: {reason}"
  5. Polling: frontend should poll GET `/api/cron-jobs/:id/logs?limit=1` every 3 seconds for up to 30 seconds to update the last result status from "running" to actual result

### 6. Run Log Viewer

- **Trigger:** Click "Logs" button or a chevron to expand the row
- **Pattern:** Expandable sub-row below the job row (preferred) or modal dialog
- **Expanded sub-row content:**
  ```
  ┌─────────────────────────────────────────────┐
  │ ▲ {job_name}            [Close]             │
  │─────────────────────────────────────────────│
  │ Run Time          Status    Result          │
  │ 2026-05-27 10:00  OK        Collected 42    │
  │ 2026-05-27 09:00  Error     Timeout after   │
  │ 2026-05-27 08:00  OK        Collected 38    │
  │ ...                         30s             │
  │                                             │
  │ [Show more...]                              │
  └─────────────────────────────────────────────┘
  ```
- Columns in log table: started_at, finished_at, duration (computed), status (badge), result_summary (truncated to 1 line with ellipsis), error_message (red text, only shown if status=Error)
- Loads last 20 logs on expand, with "Show more..." pagination
- Loading state: skeleton or "Loading logs..." text in the expanded area
- Empty state: "No execution history for this job yet."

### 7. Save/Cancel for Expression Edit

- **Save:** PATCH `/api/cron-jobs/:id` with `{ cron_expr: newValue }`
- **Success:** Input exits edit mode, cell shows new value, brief green flash on the cell
- **Error:** Input border turns `--danger`, show inline error "Invalid cron expression" or "Failed to save: {reason}"
- **Cancel:** Press Escape or click X button — revert to original value

### 8. Error & Edge Case Handling

| Situation | Treatment |
|-----------|-----------|
| API load fails (network error) | Centered error state: "Failed to load cron jobs. [Retry]" button |
| Single job toggle fails | Show toast at top-right: "Failed to update {job_name}" |
| Save expression fails (validation) | Inline error below input, border turns `--danger` |
| Save expression fails (server error) | Toast: "Failed to save expression" |
| Manual trigger fails | Error message in confirmation dialog, button re-enables |
| Log load fails | Inline error in expanded area: "Failed to load logs. [Retry]" |
| Very long job names | Ellipsis truncation (`text-overflow: ellipsis`) |
| Very long result summaries | Single-line truncation in table, full text in log viewer |

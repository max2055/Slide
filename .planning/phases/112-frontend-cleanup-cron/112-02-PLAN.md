## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP API client → cron API routes | Untrusted API consumer making requests to /api/cron/* |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-112-03 | Tampering | PUT/POST cron API | mitigate | `requirePermission('cron:manage')` on all write routes; JWT auth on all routes |
| T-112-04 | Information Disclosure | GET cron API | mitigate | `requirePermission('cron:view')` on read routes; JWT auth required |
| T-112-05 | DoS | Cron expression validation | mitigate | try-catch `new CronJob(expr, () => {})` before persisting to DB; reject invalid expressions with 400 |
| T-112-06 | Tampering | SQL injection | mitigate | All queries use parameterized `pool.execute('?', [val])` — no string concatenation |
| T-112-07 | Elevation of Privilege | Permission code assignment | mitigate | `cron:manage` only assigned to admin (* wildcard) and dba roles, per D-20 |
| T-112-08 | Tampering | Concurrency | mitigate | CronManager uses `runningFlags` Set to skip overlapping executions; per RESEARCH.md Pitfall 5 |
| T-112-SC | Tampering | npm installs | mitigate | No new packages in this plan. `cron` v4.4.0 already installed and verified by research. |
</threat_model>

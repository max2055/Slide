1. 仪表盘

仪表盘是登录后的默认首页，提供系统概览视图。

主要功能

  • 实例数量统计: 按数据库类型（MySQL、PostgreSQL、Oracle）展示实例分布
  • 告警统计卡片: 当前活跃告警数和各级别告警分布
  • 最新 AI 分析摘要: 最近完成的 AI RCA 分析和诊断结果
  • 容量趋势图: 基于历史数据的容量预测趋势线

操作步骤

  1. 登录系统，自动跳转到仪表盘
  2. 顶部的统计卡片展示实时指标概览
  3. 查看容量趋势图了解实例增长情况
  4. 点击任意统计卡片可跳转到对应的详情页面

2. 实例管理

实例管理是系统的核心功能模块，负责纳管所有数据库实例的配置、状态监控和连接管理。

实例列表

首页展示所有已纳管的数据库实例列表，包含实例名、类型、主机地址、端口、健康状态等信息。

添加实例

  1. 点击右上角"添加实例"按钮
  2. 选择数据库类型（MySQL / PostgreSQL / Oracle）
  3. 填写连接信息：主机地址、端口、用户名、密码、默认数据库
  4. 点击"测试连接"按钮验证配置正确性
  5. 连接成功后点击"保存"

实例详情

点击实例列表中的实例名进入详情页，包含多个子标签页：

  • 概览: 实例基本信息和健康状态
  • 监控: 关键性能指标的时间序列图表（连接数、QPS、TPS、CPU、内存、IO）
  • TopSQL: 按执行时间排序的慢查询列表
  • QAN: 查询分析（Query Analytics）页面
  • 执行计划: Explain 执行计划可视化
  • 慢查询: 慢查询日志列表
  • 容量: 数据库容量使用趋势
  • 会话: 当前数据库连接会话列表

3. SQL 控制台

SQL 控制台提供在浏览器中编写和执行 SQL 语句的能力，集成了 CodeMirror 编辑器，支持语法高亮和自动补全。

使用 SQL 控制台

  1. 在左侧导航栏进入"SQL 控制台"
  2. 从下拉列表中选择目标实例和数据库
  3. 在编辑器中输入 SQL 语句（支持语法高亮、自动补全）
  4. 点击"执行"按钮（或 Ctrl+Enter）运行 SQL
  5. 在下方结果表格中查看返回数据

AI SQL 审核

在执行高风险 SQL 前，可以点击"审核"按钮，AI 会对 SQL 进行风险评估（权限越界、性能风险、数据泄露等）。

4. 告警管理

告警管理模块提供完整的告警生命周期管理，包括告警浏览、分析、处理和设置。

浏览告警

  1. 进入"告警"页面查看告警列表
  2. 按严重级别（warning / error / critical）、实例、时间范围筛选
  3. 点击某条告警进入详情页

AI 告警分析

已分析的告警会显示"已分析"标记，点击可查看 RCA（根因分析）和故障诊断结果。

告警设置

  • 告警升级规则: 配置超时自动升级到更高级别
  • 维护窗口: 在计划变更期间自动静默告警
  • 静默规则: 临时静音特定指标或实例的告警

告警事件管理

每条告警都有完整的事件记录：创建 → 分配 → 调查 → 备注 → RCA 关联 → 解决 → 关闭 → 事后复盘。

5. AI 分析

AI 分析模块利用 LLM 对数据库问题进行智能诊断和分析。

分析类型

  • RCA（根因分析）: 对告警进行深度根因分析
  • TopSQL 分析: 对慢查询进行优化建议分析
  • 故障诊断: 对不健康实例执行全栈诊断

自动分析

当 ENABLE_AUTO_AI_ANALYSIS=true 时：

  • 每 10 秒检查新慢查询 → 触发 TopSQL 分析
  • 每 10 秒检查新告警 → 触发 RCA 分析
  • 每 60 秒检查不健康实例 → 触发故障诊断

手动分析

在告警详情页点击"分析"按钮手动触发 AI 分析。

6. Chat 助手

AI Chat 是 Slide 的智能交互入口，支持自然语言问答和数据库运维操作。

使用 Chat

  1. 在左侧导航栏进入"Chat"页面
  2. 在输入框中输入数据库运维相关问题
  3. 支持的问题类型："哪些实例有问题"、"最近的慢查询"、"当前告警状态"等
  4. AI 实时流式回复，回复中可能包含可点击的回链导航到相关页面

Agent 选择器

如有多个 Agent，可在 Chat 界面顶部切换 Agent。

报表模块提供多种类型的数据库运维报表生成和导出。

报表类型

报表说明格式

8. SQL 审核审批

审批模块为高风险 SQL 操作提供人工审批流程保障。

操作步骤

  1. 在 SQL 控制台中提交 SQL 审核请求
  2. 审批人在"审批"页面查看待审批列表
  3. 审查 AI 风险评价和 SQL 内容
  4. 选择"通过"或"拒绝"，可添加审批意见
  5. 支持批量审批操作

9. RBAC 权限管理

基于角色的访问控制（RBAC）提供三层权限体系。

组件说明

  • 用户管理: 创建、编辑、删除用户
  • 角色管理: 定义角色并分配权限点
  • 实例级访问控制: 控制用户对特定数据库实例的访问权限

操作步骤

  1. 管理员进入"用户管理"页面创建用户
  2. 进入"角色管理"页面定义角色并分配权限点
  3. 将用户绑定到相应角色
  4. 配置用户可访问的数据库实例列表

10. AI 设置

AI 设置页面提供 AI 分析功能的全局配置。

配置项说明

管理 AI 分析的触发条件和范围，避免不必要的 LLM 调用消耗。

配置操作

  1. 进入"设置" → "AI 分析"
  2. 配置：全局启用开关（开启/关闭自动分析）、分析严重级别（warning/error/critical）、实例白名单（限制 AI 分析的实例）、分析时间窗口（如 09:00-18:00）
  3. 点击"保存"，配置立即生效

12. 常见问题 (FAQ)

Q: 如何添加新的数据库实例？

A: 进入"实例管理"页面，点击右上角的"添加实例"按钮。填写数据库类型、主机地址、端口、用户名、密码和数据库名，点击"测试连接"验证配置正确性，确认连接成功后点击"保存"。

Q: AI 分析结果出现在哪里？

A: AI 分析结果有两个入口：在告警列表中，已触发 AI 分析的告警会显示"已分析"标记，点击可查看 RCA 结果；在实例详情页的诊断 tab 中，可查看该实例相关的分析摘要。

Q: 为什么 Chat AI 没有反应？

A: 可能的原因：(1) LLM API Key 未配置或配置错误 — 检查 .env 中 ANTHROPIC_API_KEY 或 OPENAI_API_KEY；(2) Gateway 未正常运行 — 检查后端启动日志；(3) Gateway WebSocket 连接失败 — 确认 28789 端口未被占用。

Q: ENABLE_AUTO_AI_ANALYSIS 有什么用？

A: 将 .env 中的 ENABLE_AUTO_AI_ANALYSIS 设为 true 后，系统会自动对新告警和新慢查询触发 AI 分析，消耗 LLM token。建议生产环境结合实例白名单和时间窗口使用。

Q: 如何在已有实例上查看数据库列表？

A: 在"SQL 控制台"页面，选择目标实例后在下拉选择器中查看数据库列表。也可以直接在编辑器中执行 SHOW DATABASES; 或 SELECT schema_name FROM information_schema.schemata;。

Q: 如何修改用户密码？

A: 管理员进入"用户管理"页面，找到对应用户，点击编辑后设置新密码。普通用户登录后可在个人设置中修改自己的密码。

Q: 如何配置通知渠道？

A: 在告警设置页中配置钉钉、企业微信、飞书或 Webhook 的通知目标。通知渠道配置完成后请联系管理员启用 notificationService。

Q: 报表导出格式有什么区别？

A: PDF: 适合正式报告和分享，格式固定；HTML: 适合在浏览器中查看，保留交互性；JSON: 适合程序化处理和二次分析；Markdown: 适合嵌入其他文档系统。

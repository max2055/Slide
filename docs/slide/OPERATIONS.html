Slide 运维文档

版本: v1.2  |  目标读者: 运维人员  |  最后更新: 2026-05-17

本文档涵盖 Slide 数据库运维平台的完整运维流程，包括系统部署、配置、启动停止、Gateway 运行机制、健康检查和故障排查。

系统概览

Slide v1.2 由多个服务组件构成，各组件通过不同的网络端口提供服务和通信。

组件技术栈端口作用
后端 APIFastify + TypeScript3000 (默认)REST API 服务
前端（开发）Lit 3.3 + Vite5173 (默认)开发热重载服务器
前端（生产）Lit 3.3 (静态构建)由 Web 服务器决定Vite 构建后的静态文件
OpenClaw GatewayNode.js + ws28789WebSocket 服务，AI Agent 消息路由
MySQLMySQL 83306主业务数据库
ElasticsearchES 89200 (默认)日志存储和查询分析（可选）
RedisRedis 76379 (默认)缓存和会话存储（可选）
MongoDBMongoDB 727017 (默认)柔性数据存储（可选）

端口来源: 端口通过 .env 文件中的 BACKEND_PORT（默认 3000）、FRONTEND_PORT（默认 5173）配置。Gateway 端口在 server.ts 中硬编码为 28789（第 3044 行）。

生产环境架构

生产环境下建议将各组件分离部署：

  • 后端 API + Gateway 部署在同一台服务器
  • 前端构建为静态文件后部署到 Nginx / CDN
  • MySQL、ES、Redis、MongoDB 各自独立部署或使用托管服务
  • 生产环境必须修改默认 JWT 密钥和加密密钥

启动流程

首次部署完整步骤

第 1 步：确保依赖服务运行

确认 MySQL 服务已启动并可连接：

mysql -h localhost -u root -p -e "SELECT 1"

第 2 步：创建并配置环境变量

cd apps/db-ops-api
cp .env.example .env

编辑 .env，至少配置：DB_HOST、DB_PORT、DB_USER、DB_PASSWORD、DB_NAME、JWT_SECRET_KEY、ANTHROPIC_API_KEY

第 3 步：安装依赖

cd apps/db-ops-api && npm install
cd frontend && npm install

第 4 步：初始化数据库

cd apps/db-ops-api && npx tsx init-db.ts

第 5 步：启动后端

cd apps/db-ops-api && npx tsx server.ts

启动流程：加载 .env → DB 初始化 → LLM 初始化 → CORS → 注册路由 → 启动 Gateway (:28789) → 启动监控采集 (30s) → 启动告警评估 (60s)

第 6 步：启动前端

cd frontend && npm run dev

第 7 步：访问和登录

浏览器访问 http://localhost:5173

用户名密码
adminTpam1234

Gateway 运行机制

概述

OpenClaw Gateway 是 Slide AI 功能的核心通信层。在 server.ts 第 3044 行启动，绑定 ws://127.0.0.1:28789/ws。

WebSocket 连接生命周期

  1. 连接: 客户端 WebSocket 连接 /ws 端点
  2. 认证: 发送 connect 消息，携带 JWT 令牌
  3. 会话创建: Gateway 创建会话上下文
  4. 消息交换: 客户端发送 request → Agent 处理 → 流式推送回复
  5. 断开: 客户端断开或超时，清理会话资源

默认 Agent: "Slide Database Assistant"，由 openclaw-runtime.ts 配置：

故障排查

问题可能原因解决方案
后端启动失败：数据库连接失败MySQL 未运行或 .env 配置错误检查 MySQL 服务，验证 .env 中 DB_* 配置
端口被占用:3000 或 :28789 被占用lsof -ti:3000 | xargs kill -9
前端白屏后端未启动或 CORS 错误先启动后端再启动前端
登录失败密码错误或数据库未初始化确认已执行 init-db.ts
Chat AI 无响应LLM API Key 未配置或无效检查 ANTHROPIC_API_KEY / OPENAI_API_KEY
Gateway 连接失败Gateway 未启动或端口错误确认日志中"OpenClaw Gateway 已启动"
告警未触发告警引擎未运行或规则未配置检查日志和 UI 告警设置
自动 AI 分析未执行ENABLE_AUTO_AI_ANALYSIS 未设为 true设置 ENABLE_AUTO_AI_ANALYSIS=true 并重启

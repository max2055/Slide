## 1. 仪表盘

仪表盘是登录后的默认首页，提供系统运行状况的整体概览。

### 功能简介

仪表盘展示以下关键信息：

- **实例统计**: 所有纳管数据库实例的总数及健康/警告/严重状态分布
- **告警统计**: 总告警数、未读告警数、严重和警告告警数量
- **AI 分析摘要**: 最新 AI 分析结果概览
- **容量趋势图**: 基于历史数据的容量预测趋势

### 操作步骤

1. 在浏览器中访问前端地址（默认 `http://localhost:5173`）
2. 使用管理员账号登录（默认 `admin` / `Tpam1234`）
3. 登录后自动跳转到仪表盘页面
4. 查看顶部统计卡片：实例总数、健康实例数、警告实例数、严重实例数、告警总数、未读告警数
5. 点击任意统计卡片可跳转到对应的详情页面
6. 查看容量趋势图表区域，了解实例资源的增长趋势

## 2. 实例管理

实例管理模块用于纳管数据库实例，支持 MySQL、PostgreSQL、Oracle 等多种数据库类型。

### 实例管理功能

实例管理提供数据库实例的完整生命周期管理，包括添加、编辑、测试连接、查看详情和删除。每个实例详情页包含多个子页面用于深入分析。

### 实例操作步骤

#### 添加实例

1. 在左侧导航栏点击"实例管理"进入实例列表页
2. 点击页面右上角的"添加实例"按钮
3. 在弹出的表单中选择数据库类型（MySQL / PostgreSQL / Oracle）
4. 填写连接信息：主机地址、端口、用户名、密码、数据库名
5. 点击"测试连接"验证配置正确性
6. 确认连接成功后点击"保存"
7. 新实例将出现在实例列表中

> ![添加实例表单](assets/screenshots/instance-add-form.png)
> 截图待补充：添加实例表单

#### 查看实例详情

1. 在实例列表中点击任意实例进入详情页
2. 详情页包含以下子页面（tab 切换）：
   - **概览**: 实例基本信息、当前状态、性能概览
   - **指标**: CPU、内存、连接数、IO 等时间序列图表
   - **TopSQL**: 按资源消耗排序的 SQL 语句
   - **QAN**: 查询分析（需要 Elasticsearch）
   - **执行计划**: EXPLAIN JSON 可视化
   - **慢查询**: 慢查询列表及详情
   - **容量**: 容量预测与分析
   - **会话**: 当前活跃会话列表

## 3. SQL 控制台

SQL 控制台提供在线数据库查询和 SQL 执行能力，集成 AI 审核功能。

### SQL 控制台功能

- **SQL 编辑器**: 基于 CodeMirror 的专业编辑器，支持语法高亮、自动补全
- **查询执行**: 选择实例和数据库后执行 SQL
- **结果查看**: 以表格形式展示查询结果
- **查询历史**: 记录所有已执行查询
- **AI SQL 审核**: 提交前由 LLM 检测 SQL 风险
- **SQL 审批流**: 高风险 SQL 需审批后执行

### 控制台操作步骤

#### 执行查询

1. 在左侧导航栏点击"SQL 控制台"
2. 在页面顶部的选择器中，选择目标实例和数据库
3. 在编辑器中输入 SQL 语句
4. 点击"执行"按钮或按快捷键运行
5. 查看查询结果表格
6. 需要时可导出结果数据

> ![SQL 控制台](assets/screenshots/sql-console.png)
> 截图待补充：SQL 控制台主界面

#### AI SQL 审核

1. 在编辑器中输入 SQL 后，点击"审核"按钮
2. 系统将 SQL 发送给 LLM 进行风险检测
3. 审核结果面板显示风险等级和修改建议
4. 根据审核结果决定是否执行：
   - 低风险：可直接执行
   - 中高风险：建议修改后重新审核
   - 高风险：需提交审批后执行

> ![SQL 审核面板](assets/screenshots/sql-audit-panel.png)
> 截图待补充：AI SQL 审核结果面板

#### 查询历史

1. 在 SQL 控制台的历史 tab 中查看所有已执行查询
2. 可按时间范围、实例等条件筛选
3. 点击历史记录中的 SQL 可重新加载到编辑器中

## 4. 告警管理

告警管理模块提供数据库运行异常的告警发现、分析和处理。

### 告警管理功能

告警引擎每 60 秒评估所有活跃实例的指标规则，当指标超过阈值时生成告警。告警系统支持以下功能：

- **告警列表**: 按严重级别、状态、时间筛选告警
- **告警详情**: 查看告警触发值、指标趋势、关联事件
- **AI 告警分析**: 对告警进行根因分析（RCA）
- **告警升级**: 根据升级规则定时升级未处理的告警
- **维护窗口**: 在已知变更期间静默告警
- **告警静默**: 临时静音特定告警
- **告警事件**: 事件记录、指派和调查

### 告警操作步骤

#### 查看告警列表

1. 在左侧导航栏点击"告警"进入告警列表
2. 列表显示告警标题、严重级别（critical/warning/error）、状态、来源实例、触发时间
3. 使用筛选条件按严重级别、状态、时间范围过滤
4. 已触发 AI 分析的告警显示"已分析"标记

3. 如果 AI 分析已完成，可查看 RCA 诊断结果
4. 可执行操作：标记已读、确认、触发手动 AI 分析

## 5. AI 分析

AI 分析模块提供基于 LLM 的智能数据库运维分析能力。

### 分析类型

AI 分析支持三种分析类型：

- **根因分析 (RCA)**: 分析告警的根因，给出诊断结论和修复建议
- **TopSQL 分析**: 分析慢查询的 SQL 性能问题，提供优化建议
- **故障诊断**: 综合分析故障场景，给出排查路径

### 触发方式

#### 自动触发

当 `.env` 中 `ENABLE_AUTO_AI_ANALYSIS=true` 时：

- **TopSQL 自动分析**: 每 10 秒检查新慢查询，对执行时间超过 10 秒的查询自动触发分析
- **告警 RCA 自动分析**: 每 10 秒检查新告警，自动触发 RCA 分析

#### 手动触发

1. 在告警详情页点击"分析"按钮
2. 在实例详情页的诊断 tab 中点击"执行诊断"
3. 系统调用 LLM 进行分析并返回结果

### 配置管理

AI 分析配置（在 AI 设置页中配置）：

- 全局启用开关
- 分析的严重级别筛选（warning / error / critical）
- 实例白名单
- 分析时间窗口

## 6. Chat 助手

Chat 助手提供基于 AI Agent 的交互式运维问答界面。

### Chat 功能

Chat 助手通过 OpenClaw Gateway 的 WebSocket 通道与 AI Agent 实时通信，Agent 可以：

- 查询当前系统状态（"哪些实例有问题"）
- 检索最近的慢查询
- 分析告警状态
- 执行健康检查
- 提供优化建议

### 使用步骤

1. 在左侧导航栏点击"Chat"进入会话页面
2. 在输入框中输入问题，例如：
   - "当前有哪些实例处于异常状态？"
   - "最近有哪些慢查询？"
   - "告警列表中有几个严重告警？"
3. 按回车或点击发送按钮
4. AI Agent 将以流式方式逐段生成回复
5. 回复中的链接可点击跳转到相关页面
6. 如果配置了多个 Agent，可通过 Agent 选择器切换

## 8. SQL 审核审批

SQL 审核审批模块提供高风险 SQL 执行前的审批流程。

### 审批功能

- **审批列表**: 待审批的 SQL 列表，显示提交人、提交时间、风险等级
- **审批详情**: 查看 SQL 完整内容、风险评估结果、修改建议
- **审批操作**: 通过或拒绝 SQL 执行
- **批量操作**: 勾选多条待审批 SQL 进行批量通过或拒绝
- **审批历史**: 记录所有已处理的审批

### 审批操作步骤

1. 在左侧导航栏点击"审批"进入审批列表
2. 查看待审批的 SQL 条目列表
3. 点击条目进入详情页：
   - 查看 SQL 全文和变更内容
   - 查看 AI 风险检测结果和建议
   - 查看 SQL Diff（对比原 SQL 和修改后 SQL）
4. 决定操作：
   - **通过**: SQL 将自动执行
   - **拒绝**: 填写拒绝原因，SQL 不执行
5. 批量操作：勾选多条后，选择"批量通过"或"批量拒绝"
6. 已处理的审批可在历史记录中查看

## 10. AI 设置

AI 设置页面提供 AI 分析功能的全局配置。

### 配置项说明

管理 AI 分析的触发条件和范围，避免不必要的 LLM 调用消耗。

3. 配置完成后点击"保存"
4. 配置立即生效，无需重启服务

## 12. 常见问题 (FAQ)

### Q: 如何添加新的数据库实例？

**A:** 进入"实例管理"页面，点击右上角的"添加实例"按钮。填写数据库类型、主机地址、端口、用户名、密码和数据库名，点击"测试连接"验证配置正确性，确认连接成功后点击"保存"。

### Q: AI 分析结果出现在哪里？

**A:** AI 分析结果有两个入口：

- 在告警列表中，已触发 AI 分析的告警会显示"已分析"标记，点击可查看 RCA 结果
- 在实例详情页的诊断 tab 中，可查看该实例相关的分析摘要

### Q: 为什么 Chat AI 没有反应？

**A:** 可能的原因：

1. LLM API Key 未配置或配置错误 — 检查 `.env` 中 `ANTHROPIC_API_KEY` 或 `OPENAI_API_KEY`
2. Gateway 未正常运行 — 检查后端启动日志中是否有"OpenClaw Gateway 已启动"
3. Gateway WebSocket 连接失败 — 确认 28789 端口未被占用

### Q: ENABLE_AUTO_AI_ANALYSIS 有什么用？

**A:** 将 `.env` 中的 `ENABLE_AUTO_AI_ANALYSIS` 设为 `true` 后，系统会自动对每 10 秒检查到的新告警和新慢查询触发 AI 分析。这会消耗 LLM token，建议在生产环境中结合实例白名单和时间窗口使用，避免不必要的消耗。

### Q: 如何在已有实例上查看数据库列表？

**A:** 在"SQL 控制台"页面，选择目标实例后，在查询数据库选择器中可以看到该实例下的所有数据库。也可以直接在编辑器中执行 `SHOW DATABASES;` 或 `SELECT schema_name FROM information_schema.schemata;` 来查看。

### Q: 如何修改用户密码？

**A:** 用户密码重置需要管理员操作：进入"用户管理"页面，找到对应用户，点击编辑后设置新密码。普通用户在登录后可在个人设置中修改自己的密码。

### Q: 如何配置通知渠道？

**A:** 目前通知推送服务暂不自动启动（待通知渠道配置 UI 完善），需要在告警设置页中配置钉钉、企业微信、飞书或 Webhook 的通知目标。通知渠道配置完成后请联系管理员启用 `notificationService`。

### Q: 报表导出格式有什么区别？

**A:** 四种导出格式各有适用场景：

- **PDF**: 适合正式报告和分享，格式固定
- **HTML**: 适合在浏览器中查看，保留交互性
- **JSON**: 适合程序化处理和二次分析
- **Markdown**: 适合嵌入其他文档系统

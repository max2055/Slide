Slide 项目结构

概览

Slide 是一个 AI 原生的数据库运维管理平台，基于 OpenClaw 的 Control UI 架构构建。

架构分层

1. 核心层 (openclaw/)

目的: 提供基础 UI 框架和通信协议。

关键组件:

  • OpenClawApp — Lit 主组件，含生命周期管理
  • GatewayBrowserClient — WebSocket 客户端，自动重连
  • Host 模式的状态管理
  • 主题系统和 i18n

文件数: 124 个 TypeScript 文件 (frontend/src/openclaw/ui/)

2. 业务层 (slide/)

目的: 实现 Slide 专属的业务逻辑。

服务:

  • DatabaseService — 实例管理、健康检查、指标
  • AlertService — 告警列表和确认
  • LLMService — LLM 配置和测试
  • ChatService — AI 对话操作
  • SkillsService — 技能执行
  • ReportsService — 报表生成
  • UserService — 用户管理
  • AuthService — 认证

适配器: SlideGatewayAdapter — 将 OpenClaw RPC 方法映射到 Slide REST API

3. 应用层 (pages/)

目的: 面向用户的页面和组件。

页面: DashboardPage（仪表盘概览）、InstancesPage（实例管理）、AlertsPage（告警监控）、ChatPage（AI Chat）、LLMSettingsPage（LLM 配置）、ReportsPage（报表管理）、UserManagementPage（用户管理）

数据流

用户操作 → OpenClaw App → Gateway Adapter → Slide API → 后端
              ↓                                    ↓
         Lit 组件                           Fastify Server
              ↓                                    ↓
         OpenClaw Core  ←  Gateway 协议  ←  Database

开发工作流

启动开发

# Terminal 1 - 后端
cd apps/db-ops-api
npm run dev

# Terminal 2 - 前端
cd frontend
npm run dev

新增功能

  1. 定义类型 → slide/types/index.ts
  2. 添加服务方法 → slide/services/index.ts
  3. 注册 RPC 处理器 → slide/adapters/gateway-adapter.ts
  4. 创建/更新页面 → frontend/src/pages/
  5. 更新导航 → frontend/src/openclaw/ui/navigation.ts

关键决策

  1. Fork 模式: 维护 OpenClaw 的自定义分支而非直接复制文件
  2. 三层架构: 核心层、业务层、应用层清晰分离
  3. Gateway 适配器模式: 将 RPC 翻译为 REST 实现干净集成
  4. Host 模式状态管理: 遵循 OpenClaw 的状态管理模式
  5. 全栈 TypeScript: 所有层都使用完整类型安全

现有页面保留在 frontend/src/pages/，API 客户端在 frontend/src/api/ 继续可用。
